# MVP品質チェックスキル

指定されたMVPアプリの品質を5軸100点で採点し、改善計画を策定します。

## 入力
$ARGUMENTS
（例: "payment-alert" や "crosssell doc-checker" や "全アプリ"）

## 採点基準（100点満点・5カテゴリ x 20点）

### 1. 機能完全性（20点）
| 点数 | 基準 |
|------|------|
| 18-20 | 全機能が正確に動作、エッジケース対応済み |
| 14-17 | 主要機能OK、軽微なロジック問題あり |
| 10-13 | 一部機能が未実装または不正確 |
| 0-9 | 重大なバグ、機能欠落 |

チェック項目:
- データ処理ロジックの正確性
- フィルター・ソート機能の動作
- CSV読込/ダウンロード機能
- サンプルデータ自動読込
- エラー時の挙動

### 2. ビジネス訴求力（20点）
| 点数 | 基準 |
|------|------|
| 18-20 | 導入効果が定量的・具体的、業界用語が正確 |
| 14-17 | 訴求ポイントあり、一部抽象的 |
| 10-13 | 一般的な表現、業界特化度が低い |
| 0-9 | ビジネス訴求なし |

チェック項目:
- 導入効果の具体的数値（工数削減○時間→○分、年間¥○○万）
- 業界用語・専門用語の正確性
- KPIカードの指標選定が適切か
- ターゲットユーザーの課題に直結しているか

### 3. UX/デザイン（20点）
| 点数 | 基準 |
|------|------|
| 18-20 | 初見で迷わず使える、フィルター/ダウンロード完備 |
| 14-17 | 基本的に使いやすい、小改善余地あり |
| 10-13 | 操作に迷う箇所がある |
| 0-9 | 操作性に重大な問題 |

チェック項目:
- タブ構成の直感性
- フィルター・絞り込み機能
- ダウンロード（CSV/テキスト）機能
- サイドバーの情報量と整理度
- エラー時のユーザーガイド

### 4. デモインパクト（20点）
| 点数 | 基準 |
|------|------|
| 18-20 | 開いた瞬間に「おっ」となる、グラフ・KPIが映える |
| 14-17 | 見栄えは良い、もう一工夫でインパクト向上 |
| 10-13 | 情報は表示されるが地味 |
| 0-9 | 視覚的訴求力なし |

チェック項目:
- Hero部分のビジュアル
- KPIカードの数値インパクト
- グラフの種類・色使い・タイトル
- サンプルデータの質（リアリティ・件数）
- 初期表示時の情報量（空画面でないか）

### 5. コード品質（20点）
| 点数 | 基準 |
|------|------|
| 18-20 | 構文OK、パターン統一、self-contained完全 |
| 14-17 | 構文OK、軽微なパターン不統一 |
| 10-13 | 動作するが構造に問題あり |
| 0-9 | 構文エラー、外部依存 |

チェック項目:
- `ast.parse()` 通過
- self-contained（外部モジュール非依存）
- パターン統一（CSS/フッター/サイドバー）
- 数値フォーマット統一（¥表記・%表記）
- session_state管理

## 実行手順

### Step 1: コード読み込み
対象アプリの `app.py` と `create_sample_data.py` を読み込む。

### Step 2: 5軸採点
各カテゴリについて:
1. チェック項目を1つずつ評価
2. 具体的な問題点を列挙
3. 点数を決定（根拠付き）

### Step 3: 改善計画
- 減点された項目ごとに具体的な改善タスクを定義
- 改善後の想定スコアを記載
- 4エージェント構成の改善計画を提案

### Step 4: 出力フォーマット
```
## 採点結果: {アプリ名}

| カテゴリ | 点数 | 主な減点理由 |
|---------|:----:|------------|
| 機能完全性 | ○○ | ... |
| ビジネス訴求力 | ○○ | ... |
| UX/デザイン | ○○ | ... |
| デモインパクト | ○○ | ... |
| コード品質 | ○○ | ... |
| **合計** | **○○** | |

## 改善計画
| # | タスク | 対象カテゴリ | 期待スコア向上 |
|---|--------|------------|:------------:|
| 1 | ... | ... | +○ |
```

ユーザーの承認後、`/mvp-improve` で改善を実行する旨を案内する。

## 注意事項
- コードの読み込みのみ行い、この段階では編集しない
- 回答は日本語で行う
