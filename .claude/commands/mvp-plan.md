# MVP企画・優先順位付けスキル

ユーザーの入力に基づき、士業・EC事業者向けのMVPアプリ候補を整理し、優先順位を付けます。

## 入力
$ARGUMENTS
（例: "EC向け在庫管理ツール、顧客LTV分析、レコメンドエンジン" や "士業向けで追加すべきMVPを提案して"）

## 実行手順

### Step 1: 現状の棚卸し
`apps/` ディレクトリを確認し、既存MVPの一覧を把握する。

### Step 2: 候補リストの作成
ユーザー入力＋既存MVPとの重複チェック＋市場ニーズを考慮し、候補を列挙する。

各候補には以下を記載:
| 項目 | 内容 |
|------|------|
| MVP名 | 日本語名称 |
| 概要 | 1行でツールの目的を説明 |
| ターゲット | 士業 / EC / 共通 |
| 想定ユーザーの課題 | 具体的な業務ペイン |
| 差別化ポイント | 既存ツールとの違い |
| 技術スタック | Streamlit + pandas / LightGBM / etc. |
| 実装難易度 | S（1日）/ A（2-3日）/ B（1週間） |

### Step 3: 優先順位付け
以下の4軸で各候補をスコアリング（各5点、合計20点）:

1. **緊急性**: ユーザーが今すぐ欲しいか（5=今日にでも / 1=いつかは）
2. **収益インパクト**: 導入先の売上・工数削減への貢献度
3. **実装容易性**: 既存パターンの流用度、追加ライブラリの有無
4. **デモ映え**: 初見のインパクト、営業デモでの説得力

### Step 4: 要件定義（上位3つ）
優先度上位3つについて、以下のフォーマットで要件を定義:

```
## MVP名: ○○○
- アプリ名（ディレクトリ名）: apps/xxx/
- テーマカラー: #XXXXXX
- タブ構成: タブ1「○○」/ タブ2「○○」/ タブ3「○○」
- サンプルデータ: ○件 × ○カラム
  - カラム一覧: ...
- KPIカード: ○○ / ○○ / ○○
- 必須グラフ: 円グラフ / 棒グラフ / 折れ線 etc.
- CSVダウンロード: あり
- 導入効果（コピー）: 「○○を△△に改善（年間¥○○万相当）」
```

### Step 5: 出力
最終的な優先順位表（Markdown）をユーザーに提示し、承認を求める。
実装は `/mvp-build` スキルで行う旨を案内する。

## 注意事項
- この段階ではコードは書かない（計画のみ）
- 既存MVPとの機能重複を避ける
- 士業向け・EC向けの市場特性を考慮する
- 回答は日本語で行う
