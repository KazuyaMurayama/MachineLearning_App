# MVP改善実行スキル

品質チェック結果に基づき、4つのAIエージェントを並列起動して改善を実行します。

## 入力
$ARGUMENTS
（例: "payment-alert 75点→88点" や "全Tier A" や 改善計画テキスト）

## 4エージェント構成（標準）

### Agent A: 機能＋ロジック修正
**担当**: 指定MVPのデータ処理ロジック・計算の修正
- バグ修正、ロジック改善
- フィルター・ソート機能の追加
- KPIカード計算の修正
- 前月比delta表示の追加

### Agent B: UX改善
**担当**: 操作性・情報設計の改善
- サイドバーのガイド追加
- フィルター追加（業種/ステータス等）
- ダウンロード機能の拡充
- 情報階層の整理
- 初期表示の改善

### Agent C: デモインパクト強化
**担当**: 視覚的訴求力・営業デモ効果の向上
- グラフの追加・改善
- 個別カルテ・詳細ビューの追加
- トークスクリプト・テンプレートの追加
- ランク・スコアの可視化
- サマリーセクションの強化

### Agent D: 横断品質統一
**担当**: 全アプリ横断のパターン統一＋データ品質
- 構文チェック（全アプリ `ast.parse()`）
- CSS/フッター/サイドバーパターン統一
- サンプルデータの現実性検証・改善
- 顧問先名の重複排除
- 数値フォーマット統一

## 実行手順

### Step 1: 改善計画の確認
品質チェック結果（`/mvp-quality` の出力）を確認し、各エージェントのタスクを具体化する。

### Step 2: エージェント起動
Agent toolで4エージェントを**並列起動**する。

各エージェントのプロンプトには以下を含める:
1. 対象ファイルの絶対パス
2. 具体的な改善タスク（5項目以内）
3. 修正後の品質基準
4. self-contained制約の遵守指示
5. 構文チェック＋commit＋push指示

**重要**: Agent Dはapp.pyを直接編集しない（Agent A-Cとのコンフリクト回避）。
Agent Dは `create_sample_data.py` と `sample_data/*.csv` のみ編集可。

### Step 3: 結果集約
全エージェント完了後:
1. `git log --oneline -10` でコミット確認
2. 全アプリの構文チェック再実行
3. 改善後の再採点（5軸100点）

### Step 4: 再採点の出力
```
## 改善後スコア

| MVP | 改善前 | 改善後 | 改善幅 |
|-----|:-----:|:-----:|:-----:|
| ... | ○○点 | ○○点 | +○○ |

## Agent別改善サマリー
| Agent | 実施内容 | 影響カテゴリ |
|-------|---------|------------|
| A | ... | 機能 |
| B | ... | UX |
| C | ... | デモ |
| D | ... | コード品質 |
```

## 目標スコア
- 改善前スコアが70点台 → **88点以上**を目標
- 改善前スコアが80点台 → **93点以上**を目標

## 注意事項
- 1エージェント1ファイル原則（コンフリクト回避）
- 各エージェントは独立してcommit+push
- self-contained制約を絶対に破らない
- 回答は日本語で行う
