## 分析手法

3つの専門AIエージェントチームが独立して分析し、結果を統合した。

| チーム | 分析内容 | 追加評価軸 |
|--------|---------|-----------|
| **GTM戦略チーム** | Go-to-Market速度、顧客獲得容易性、横展開ポテンシャル | 3軸（各1-5点） |
| **リスク分析チーム** | 規制、プライバシー、ベンダーロック、データ品質、期待値管理、サポートコスト、価格感度、景気感応度 | 8軸（各1-5点、5=低リスク） |
| **ポートフォリオ戦略チーム** | 集中vs分散戦略、フェーズ別計画、ビジネスモデル、やらない判断 | 定性分析 |

## 推奨戦略：「製造業集中 → 建設へ隣接展開」

### 戦略選択の根拠

3チームの分析結果を統合した結果、**「1業界集中 + 隣接展開」のハイブリッド戦略**を推奨。

| 評価軸 | 製造（中小） | 建設 | EC大手 | 不動産 |
|--------|:---:|:---:|:---:|:---:|
| 競合強度 | 低〜中 | **極低（代替皆無）** | **高** | 中 |
| 意思決定速度 | **速い** | やや遅い | 遅い（稟議） | 速い |
| データ準備度 | **Excel既存** | やや低い | 高い | 中 |
| SHAP価値 | **極めて高い** | 高い | 中 | 中 |
| プライバシーリスク | **低い** | **低い** | 中 | 中 |
| 単価許容度 | 月3-10万 | 月5-20万 | 月20万〜 | 月3-10万 |

**最も重要な判断：EC大手・小売大手・エネルギーは当面やらない。** 大企業向け・専門領域は競合が強く営業サイクルが長い。製造・建設という「DXが最も遅れている × 代替ツールがない × Excelデータが眠っている」セグメントにリソースを集中。

## 戦略的シナジーマップ

### 同一企業内クロスセル（最強のシナジー）

```
製造業の1社に対して：
  見積もり金額精緻化 → 歩留まり予測 → リードタイム予測 → 残業時間予測
  ───────────────────────────────────────────────────
  営業部門          製造部門       生産管理       人事部門

  ※同じCSVデータソース（生産管理システム出力）を使い回せる
  ※SHAP解析結果が部門横断の改善会議で共有 → 社内口コミ効果
```

### パッケージ化案

| パッケージ | 含まれるユースケース | 月額 |
|-----------|-------------------|------|
| **製造業DXパック** | 見積もり精緻化 + 歩留まり予測 + リードタイム予測 | **10万円**（単体計15万円） |
| **建設プロジェクト管理パック** | 工事原価予測 + 工期遅延予測 | **12万円** |
| **全社コスト管理パック** | 上記 + 残業時間予測 | **+3万円** |

### サプライチェーン横断シナジー

```
製造業（既存顧客）──紹介──→ 物流会社（新規）
  リードタイム予測              受注物量予測
  ※紹介経由のCACはほぼゼロ
```

### LTVモデル

```
月1-3  ：見積もり精緻化で導入（月5万円）
月4-6  ：歩留まり予測を追加（→月10万円）
月7-12 ：リードタイム+残業予測を追加（→月13万円）
月13-24：API連携・カスタムダッシュボード（→月20万円）

LTV = 20万円 × 24ヶ月 × 0.85（チャーン考慮） ≈ 408万円
CAC目標 = LTVの1/3 ≈ 136万円以下
```

## 「やらない」判断リスト

| ユースケース | 見送り理由 | 再検討タイミング |
|------------|-----------|----------------|
| EC（大）: 販売数量予測 | Amazon Forecast等の競合が強力。大企業の稟議が6ヶ月超 | ARR 1億円超で大企業営業チーム構築後 |
| EC（大）: 顧客LTV予測 | CDPベンダー（Treasure Data等）が統合機能として提供 | 同上 |
| EC（大）: 返品率予測 | 同上 | 同上 |
| SaaS（大）: MRR予測 | SaaS企業は自社でPython内製する傾向。ノーコードの価値薄い | PLG戦略でFree層に取り込めた場合 |
| 小売（大）: 需要予測 | 日立・NEC等の大手SIerが占有。参入障壁極めて高い | 中小小売にピボットする場合 |
| エネルギー（2件） | 気象データ連携必須。ドメイン知識の獲得コスト高い | 業界パートナーが見つかった場合 |
| 物流（大）: 倉庫人員予測 | WMS連携が前提。API開発必要 | API機能実装後 |

**Top 20のうち8件を見送り、12件に集中** → 開発リソースの70%を製造・建設に投下。

## 追補: Step 6-7レビューを反映した改訂

以下は、Step 5〜7の論理的思考・批判的思考・横断的思考レビューで指摘された課題を反映した改訂事項である。

### Tier分類の境界基準の明確化

Step 5の論理的思考レビューにおいて、Tier分類の境界が曖昧であると指摘された。以下の明確な基準を設定する。

| Tier | 基準 |
|------|------|
| **Tier 1** | 統合スコア65以上 AND GTM速度4以上 AND リスク計27以上 |
| **Tier 2** | 統合スコア60-64 OR GTM速度3 |
| **Tier 3** | 統合スコア59以下 OR 機能拡張が必要 |

この基準により、現行のTier分類における主観的判断を排除し、再現可能な分類を実現する。なお、複数条件に該当する場合は低いTier（より慎重な分類）を適用する。

### 統合スコアの重み付け問題

Step 5の批判的思考レビューにおいて、リスクスコア（40点満点）が統合スコア85点中の47%を占め、低リスク項目に偏重するバイアスが生じていると指摘された。

**リバランス後の計算式：**

```
リバランススコア = Step2(30pt) × 1.0 + GTM(15pt) × 1.5 + Risk(40pt) × 0.75
最大値 = 30 + 22.5 + 30 = 82.5pt（≒75ptに丸め）
```

**リバランス後のTop 5比較：**

| 順位 | ユースケース | 現行スコア | リバランススコア | 順位変動 |
|:---:|------------|:---:|:---:|:---:|
| 1 | 物流（中）: 受注物量予測 | 70 | 26 + 19.5 + 23.3 = **68.8** | → 維持 |
| 2 | 製造（小）: 見積もり精緻化 | 68 | 26 + 18.0 + 22.5 = **66.5** | → 維持 |
| 3 | EC（大）: 返品率予測 | 66 | 25 + 19.5 + 21.0 = **65.5** | → 維持 |
| 4 | EC（大）: 販売数量予測 | 66 | 26 + 19.5 + 20.3 = **65.8** | ↑ 6→4 |
| 5 | 物流（小）: 運賃適正価格予測 | 65 | 25 + 19.5 + 20.3 = **64.8** | → 維持 |

リバランスにより、GTM速度が高い項目（EC販売数量予測等）の順位が上昇する傾向がある。ただし、Top 3の顔ぶれに変動はなく、現行の戦略推奨（製造業集中）への影響は限定的である。

### 戦略オプションEの追加: 分析レポート販売モデル

Step 7の論理的思考・横断的思考レビューにおいて、PMF検証コストを最小化する戦略オプションとして以下が提案された。

**概要：** ツール自体のサブスクリプション販売ではなく、月1回の分析レポート（PDF 10ページ程度）を5万円で販売するモデル。

| 項目 | 内容 |
|------|------|
| **提供物** | 顧客のExcelデータを預かり、予測結果＋SHAP解析をPDFレポートで納品 |
| **価格** | 1回5万円（月次契約で月4万円） |
| **顧客体験** | ツールに触る必要なし。データを送るだけ |
| **MVP要否** | 不要（現行のStreamlitアプリを分析者が操作すれば足りる） |
| **検証コスト** | ほぼゼロ（人件費のみ） |

**既存戦略（製造業集中）との関係：**

- 戦略オプションEは、フェーズ0（後述）におけるPMF検証手段として位置付ける
- レポート販売で顧客の反応を確認した上で、ツールのセルフサービス化（SaaS化）に移行する
- レポート購入者がそのままSaaS版の初期顧客になるため、CACをゼロに近づけられる

### フェーズ0の延長: 6週間→10週間

Step 7の批判的思考レビューにおいて、MVP完成に3〜4週間が必要であるため、当初の6週間では検証期間が2〜3週間しか残らないと指摘された。

**改訂タイムライン：**

```
フェーズ0（10週間）：PMF検証
──────────────────────────────────────────────
Week 1-2  ：レポート販売（戦略オプションE）で初期反応を確認
Week 3-6  ：MVP開発（PDF出力・日本語対応・テンプレート）
Week 7-8  ：MVP版でPoC 3-5社実施
Week 9-10 ：結果分析・Go/No-Go判定
```

| 項目 | 旧（6週間） | 新（10週間） |
|------|:---:|:---:|
| レポート販売検証 | なし | 2週間 |
| MVP開発 | 3-4週間 | 4週間 |
| PoC実施・検証 | 2-3週間 | 4週間 |
| **合計** | **6週間** | **10週間** |

### 成功/撤退基準の修正

Step 6のレビューに基づき、フェーズ1のKPIを現実的な水準に修正する。

| 指標 | 旧目標 | 新目標（M6） | 修正理由 |
|------|:---:|:---:|---------|
| MRR | 40万円 | **25万円** | 中小製造業の月額単価3-5万円を前提とすると、5社で25万円が現実的 |
| PoC→有料転換率 | 50% | **25%** | 初期フェーズでは顧客教育コストが高く、50%は楽観的すぎる |

**PMFの定義を追加：**

> 「5社中3社が"なくなったら困る"と回答」をPMF達成の定性基準とする。

この定義は、Sean Ellis の PMF サーベイ（「このプロダクトが使えなくなったらどう感じますか？」に40%以上が「非常に困る」と回答）に基づく。定量KPI（MRR・転換率）と併用することで、表面的な数値達成ではなく真のプロダクト・マーケット・フィットを判定する。
