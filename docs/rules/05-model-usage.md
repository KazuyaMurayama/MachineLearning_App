# 05: モデル使い分けルール

## 基本方針

| 用途 | モデル | 理由 |
|------|--------|------|
| 計画・分析・高度な論理推論 | **Opus** | 正確性・深さが求められる場面 |
| 実行・サブタスク・その他大部分のステップ | **Sonnet** | 速度・コスト効率 |

## 具体例

### Opus を使う場面
- アーキテクチャ設計・戦略立案
- マルチエージェント改善プロセスのオーケストレーション
- 品質レビュー・批評統合
- 複雑なデバッグ・リファクタリング
- 新しい問題領域の調査・分析

### Sonnet を使う場面
- ファイル作成・更新の実行
- Git 操作（commit, push）
- 定型的なコード変更
- ファイル検索・読み取り
- テスト実行

## Agent ツールでの指定

`Agent()` 呼び出し時に `model: "opus"` または `model: "sonnet"` を指定する。
