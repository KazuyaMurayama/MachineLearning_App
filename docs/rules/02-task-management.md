# 02: タスク管理ルール

## 基本原則

- `tasks.md`（ルート）を作成・維持し、タスクを **一元管理** する。
- 新セッション開始時に `tasks.md` を参照し、未完了タスク・優先度・進捗を即座に把握できる状態を保つ。
- タスクは **細かい単位に分割** し、完了のたびに **保存・コミット** しながら進める（途中停止リスクを最小化）。

## tasks.md の構成

```
# タスク
## 起動時チェックリスト
## 次にやる（優先度順）
## 進行中
## 完了（直近）
## 既知のリスク
```

## 運用ルール

- タスク完了時は即座に `tasks.md` を更新してコミットする。バッチ更新はしない。
- 各セッション終了時に `tasks.md` の「完了」セクションに実施内容を記録する。
- `tasks.md` と Claude Code 内部の TodoWrite ツールは並行管理。TodoWrite は作業中の進捗表示用、`tasks.md` はセッション間引継ぎ用。
