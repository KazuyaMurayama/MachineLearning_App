# 03: ファイルインデックス管理ルール

## 基本原則

- `docs/file-index.md` を作成・維持し、**毎回プロンプトを受けた時に参照** する。
- 新規ファイル作成・大幅変更時は必ず `docs/file-index.md` を更新する。

## 品質チェック項目

### 1. 網羅性

- main のすべてのファイルを記述する（ブランチ分岐は廃止、main 一本化のため）。
- ディレクトリ単位で漏れがないか定期確認。

### 2. 優先度の明示

- 同種ファイルが複数ある場合、**更新が新しいものを優先確認** とする。
- 優先度が不明確な場合はファイル名に最終更新日（`YYYY-MM-DD_xxx.md`）を含めるなど工夫する。
- 各セクションに「最終更新」情報を可能な限り付与する。

### 3. 構成

```
# ファイルインデックス
## 0. セッション引継ぎ・規約
## 1. アプリ本体
## 2. サンプルデータ
## 3. 共通モジュール
## 4. 設定ファイル
## 5. ドキュメント（sales-assets, sessions, task-improvement, rules 等）
## 6. カスタムコマンド
## 7. その他
```
