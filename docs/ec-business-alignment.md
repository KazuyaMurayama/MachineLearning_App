## 1. 事業計画のEC向けサービス構造（L1/L2/L3）

事業計画（§11C + §12B）では、明示的な "L1/L2/L3" ラベルは使用していないが、実質的な3層構造が定義されている。

### 価格層の対応表

| 層 | パッケージ名 | 月額 | 含まれるアプリ数 | 主な機能 |
|----|------------|------|----------------|---------|
| **L1（スターター）** | スターターパック | ¥125,000 | ベース3本（売上予測 + 離脱予測 + 顧客セグメンテーション）| 基礎ML予測・SHAP分析 |
| **L2（グロース）** | グロースパック | ¥170,000 | 6本（L1 + LTV予測 + 在庫ABC + アソシエーション）| 横断分析・LTV最大化 |
| **L3（プレミアム）** | プレミアムパック | ¥250,000 | 12本 + 月次レビュー + カスタマイズ | 全機能 + 伴走支援 |

構造の詳細（§11C 改善版）:
- **ベース月額**: ¥80,000（コア3アプリ + 共通データ基盤）
- **追加アプリ**: ¥15,000/月/アプリ（段階アンロック）
- **データ活用顧問**: ¥100,000/月（オプション）
- **最上位到達時ARPU**: M24で平均¥230,000（標準シナリオ）、最大¥405,000（ベース+15アプリ+顧問）

### 顧客ペインと支払能力

- **ターゲット**: EC年商1億〜100億円の中堅EC事業者（国内5,000〜10,000社）
- **ペインの核心**: CAC高騰（コスト増）+ 離脱予測不能 + DS不在 + 複数SaaS乱立コスト
- **支払能力**: 月額8万〜20万円（EC事業者はROI志向が強く、効果実証後は支払意欲高）
- **競合比較**: 個別SaaS3本契約（13万〜110万円/月）vs 当社L2（17万円/月）→ 価格優位あり

### 類似競合の価格帯

| 競合カテゴリ | 代表ツール | 月額 |
|-------------|----------|------|
| BIツール | Looker, Tableau, Power BI | 5万〜20万円 |
| 需要予測SaaS | UMWELT, FOREMAST | 5千〜50万円 |
| CRM/LTV分析 | b→dash, aimstar | 3万〜30万円 |
| 汎用MLプラットフォーム | DataRobot, MatrixFlow | 20万〜100万円 |
| GA4 | Google | 無料〜（BigQuery連携で有料化） |
| **当社ポジション** | EC横断分析 | **8万〜25万円**（空白地帯） |

**差別化軸**: GA4/Lookerが「可視化」、DataRobotが「精度」に特化するのに対し、当社は「横断分析 + SHAP説明可能性 + 逆SHAP（アクション提案）」の組み合わせで空白地帯を占める。

## 2. 現在のEC MVP 4アプリの層別評価

### 2.1 `ec-demo` — 顧客離脱予測 + 需要予測（`apps/ec-demo/app.py`）

**対応層: L1（スターターパックのコア機能）**

| 評価項目 | 状況 |
|---------|------|
| 層への適合 | L1の「離脱予測アプリ（P1）」に直接対応。事業計画で最優先とされる機能 |
| 実装品質 | LightGBM分類（離脱）+ 回帰（需要）の二本柱。TreeSHAP実装済み |
| 逆SHAP | `ACTION_TEMPLATES`で13特徴量のアクション提案を実装済み（事業計画のキラー機能） |
| ギャップ1 | LabelEncodingによる順序性バイアスあり。OneHotEncoding推奨 |
| ギャップ2 | 需要予測に時系列特徴量（ラグ・移動平均・祝日フラグ）が未実装 |
| ギャップ3 | 交差検証なし（単一split）。R²が低くても警告が出ない状態 |
| ギャップ4 | 離脱モデルと需要モデルの共通データ基盤未実装 |

**L1要求充足率: 70%**

### 2.2 `ec-dashboard` — 売上ダッシュボード（`apps/ec-dashboard/app.py`）

**対応層: L1（スターターパックのベース機能）**

| 評価項目 | 状況 |
|---------|------|
| 層への適合 | L1の「売上予測アプリ（P1）」に相当するが、現状は可視化に特化しており予測機能は不在 |
| 実装品質 | KPIカード・チャートのUI品質は高い |
| ギャップ1 | MLモデルによる将来売上予測が未実装。レポーティングツール止まり |
| ギャップ2 | SHAP分析なし。GA4/Lookerと差別化できない |
| ギャップ3 | 逆SHAP未実装。L3向けキラー機能の欠如 |
| ギャップ4 | セグメント比較が弱く、競合BIツールに劣後 |

**L1要求充足率: 45%**

### 2.3 `ec-rfm` — 顧客RFM分析（`apps/ec-rfm/app.py`）

**対応層: L2（グロースパックの追加アプリ）**

| 評価項目 | 状況 |
|---------|------|
| 層への適合 | §12B「P4: RFM分析ダッシュボード」に対応。事業計画上は優先度4 |
| 実装品質 | R/F/M軸のスコアリングと顧客セグメント可視化を実装 |
| ギャップ1 | アクション提案（逆SHAP連携）が未実装 |
| ギャップ2 | LTV予測との連携がなく、橋渡しが弱い |
| ギャップ3 | コホート分析との統合がなく、時系列推移が追えない |

**L2要求充足率: 60%**

### 2.4 `ec-ad-roi` — 広告ROI分析（`apps/ec-ad-roi/app.py`）

**対応層: L2〜L3（グロース〜プレミアムの追加アプリ）**

| 評価項目 | 状況 |
|---------|------|
| 層への適合 | §12B「P3: 広告ROI予測アプリ」に直接対応 |
| 実装品質 | チャネル別ROAS分析・予算配分シミュレーション・トレンド分析を実装 |
| ギャップ1 | LightGBMによる将来ROI予測が未実装 |
| ギャップ2 | SHAP分析なし |
| ギャップ3 | What-Ifシミュレーション（逆SHAP）が未実装 |
| ギャップ4 | 離脱予測との連携なし |

**L2要求充足率: 55%**
