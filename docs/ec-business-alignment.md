## 1. 事業計画のEC向けサービス構造（L1/L2/L3）

事業計画（§11C + §12B）では、明示的な "L1/L2/L3" ラベルは使用していないが、実質的な3層構造が定義されている。

### 価格層の対応表

| 層 | パッケージ名 | 月額 | 含まれるアプリ数 | 主な機能 |
|----|------------|------|----------------|---------|
| **L1（スターター）** | スターターパック | ¥125,000 | ベース3本（売上予測 + 離脱予測 + 顧客セグメンテーション）| 基礎ML予測・SHAP分析 |
| **L2（グロース）** | グロースパック | ¥170,000 | 6本（L1 + LTV予測 + 在庫ABC + アソシエーション）| 横断分析・LTV最大化 |
| **L3（プレミアム）** | プレミアムパック | ¥260,000 | 12本 + 月次レビュー + カスタマイズ | 全機能 + 伴走支援 |

構造の詳細（§11C 改善版）:
- **ベース月額**: ¥80,000（コア3アプリ + 共通データ基盤）
- **追加アプリ**: ¥15,000/月/アプリ（段階アンロック）
- **データ活用顧問**: ¥100,000/月（オプション）
- **最上位到達時ARPU**: M24で平均¥230,000（標準シナリオ）、最大¥405,000（ベース+15アプリ+顧問）

### 顧客ペインと支払能力

- **ターゲット**: EC年商1億〜100億円の中堅EC事業者（国内5,000〜10,000社）
- **ペインの核心**: CAC高騰（コスト増）+ 離脱予測不能 + DS不在 + 複数SaaS乱立コスト
- **支払能力**: 月額8万〜20万円（EC事業者はROI志向が強く、効果実証後は支払意欲高）
- **競合比較**: 個別SaaS3本契約（13万〜110万円/月）vs 当社L2（17万円/月）→ 価格優位あり

### 類似競合の価格帯

| 競合カテゴリ | 代表ツール | 月額 |
|-------------|----------|------|
| BIツール | Looker, Tableau, Power BI | 5万〜20万円 |
| 需要予測SaaS | UMWELT, FOREMAST | 5千〜50万円 |
| CRM/LTV分析 | b→dash, aimstar | 3万〜30万円 |
| 汎用MLプラットフォーム | DataRobot, MatrixFlow | 20万〜100万円 |
| GA4 | Google | 無料〜（BigQuery連携で有料化） |
| **当社ポジション** | EC横断分析 | **8万〜26万円**（空白地帯） |

**差別化軸**: GA4/Lookerが「可視化」、DataRobotが「精度」に特化するのに対し、当社は「横断分析 + SHAP説明可能性 + 逆SHAP（アクション提案）」の組み合わせで空白地帯を占める。

## 2. 現在のEC MVP 4アプリの層別評価

### 2.1 `ec-demo` — 顧客離脱予測 + 需要予測（`apps/ec-demo/app.py`）

**対応層: L1（スターターパックのコア機能）**

| 評価項目 | 状況 |
|---------|------|
| 層への適合 | L1の「離脱予測アプリ（P1）」に直接対応。事業計画で最優先とされる機能 |
| 実装品質 | LightGBM分類（離脱）+ 回帰（需要）の二本柱。TreeSHAP実装済み |
| 逆SHAP | `ACTION_TEMPLATES`で13特徴量のアクション提案を実装済み（事業計画のキラー機能） |
| ギャップ1 | LabelEncodingによる順序性バイアスあり（§1.4 技術負債指摘）。OneHotEncoding推奨 |
| ギャップ2 | 需要予測に時系列特徴量（ラグ・移動平均・祝日フラグ）が未実装。物流/EC需要予測では致命的 |
| ギャップ3 | 交差検証なし（単一split）。R²が低くても警告が出ない状態 |
| ギャップ4 | 離脱モデルと需要モデルの共通データ基盤（§11C 改善③）未実装 |

**L1要求充足率: 70%**（逆SHAPは先行実装だが技術負債とデータ基盤が未対応）

---

### 2.2 `ec-dashboard` — 売上ダッシュボード（`apps/ec-dashboard/app.py`）

**対応層: L1（スターターパックのベース機能）**

| 評価項目 | 状況 |
|---------|------|
| 層への適合 | L1の「売上予測アプリ（P1）」に相当するが、現状は可視化に特化しており予測機能は不在 |
| 実装品質 | KPIカード・チャートのUI品質は高い。リッチなCSS実装 |
| ギャップ1 | MLモデル（LightGBM回帰）による将来売上予測が未実装。現状はレポーティングツール止まり |
| ギャップ2 | SHAP分析なし。「なぜ売上が増減したか」の要因説明機能がなく、GA4/Lookerと差別化できない |
| ギャップ3 | 逆SHAP（「売上を10%伸ばすには？」）が未実装。L3向けキラー機能の欠如 |
| ギャップ4 | セグメント比較（チャネル別・商品カテゴリ別）が弱く、競合BIツールに劣後 |

**L1要求充足率: 45%**（可視化はできているが予測とSHAPが欠如しており層の中でも下位）

---

### 2.3 `ec-rfm` — 顧客RFM分析（`apps/ec-rfm/app.py`）

**対応層: L2（グロースパックの追加アプリ）**

| 評価項目 | 状況 |
|---------|------|
| 層への適合 | §12B「P4: RFM分析ダッシュボード」に対応。ただし事業計画上は優先度4（最低層） |
| 実装品質 | R/F/M軸のスコアリングと顧客セグメント可視化を実装 |
| ギャップ1 | RFM単体では「どのセグメントに何をすべきか」のアクション提案（逆SHAP連携）が未実装 |
| ギャップ2 | LTV予測（§12B P2）との連携がなく、RFMからLTV向上施策への橋渡しが弱い |
| ギャップ3 | コホート分析（P4）との統合がなく、時系列でのセグメント推移が追えない |
| 価格根拠 | 単体では¥15,000/月の追加アプリとして提示できるが、L1（¥80,000ベース）到達前の顧客には訴求困難 |

**L2要求充足率: 60%**（RFM集計はできているがアクション提案と他アプリ連携が欠如）

### 2.4 `ec-ad-roi` — 広告ROI分析（`apps/ec-ad-roi/app.py`）

**対応層: L2〜L3（グロース〜プレミアムの追加アプリ）**

| 評価項目 | 状況 |
|---------|------|
| 層への適合 | §12B「P3: 広告ROI予測アプリ」に直接対応。CAC高騰ペインへの重要な訴求 |
| 実装品質 | チャネル別ROAS分析・予算配分シミュレーション・トレンド分析を実装 |
| ギャップ1 | LightGBMによる将来ROI予測が未実装（現状は実績値の集計・可視化） |
| ギャップ2 | SHAP分析なし。「なぜこのチャネルのROASが高いのか」の要因分解が欠如 |
| ギャップ3 | 「広告予算をXX万円増やすとROASはどう変わるか」のWhat-Ifシミュレーション（逆SHAP）が未実装 |
| ギャップ4 | 離脱予測（ec-demo）との連携なし。「離脱リスクの高いセグメントへの広告ROI」等の横断分析不可 |

**L2要求充足率: 55%**（集計・可視化はL1相当だが、ML予測とSHAPでL2〜L3に届かない）

### L3到達に必要なキラー機能

事業計画（§3.2、§8.2）が「唯一無二」と位置づける機能が現MVPに欠如している。

**キラー機能1: 逆SHAP What-Ifシミュレーター（L3必須）**
- 「広告費をXX万増やすとROASはどう変わるか」「離脱率を5%下げるには何をすべきか」を提案
- ec-demoにはEC顧客向けACTION_TEMPLATESで基礎実装済み → ec-ad-roi / ec-dashboardへ横展開が急務
- GA4・Lookerにはない機能。競合不在の差別化ポイント

**キラー機能2: 横断分析（共通データ基盤）**
- 「離脱リスクが高い顧客セグメントへの広告ROI」「RFM上位顧客の需要予測」など、アプリをまたぐ分析
- 現状：4アプリが完全独立。データ再アップロードが毎回必要
- 共通データ基盤により「1回アップロードで全アプリ連携」を実現するとロックイン効果が生まれる

**キラー機能3: 月次自動レポート + 改善提案サマリー**
- プレミアムパック（L3）の「月次レビュー」を自動化するPDF/Slack通知
- 「先月と比べて離脱リスクが上昇した顧客セグメントTOP3」等の自然言語サマリー
- これが実装されると月次解約率を1.2%以下に抑えるロックイン効果が発生（§11C 改善③）

### 競合（GA4・Looker・DataRobot等）との差別化ポイント

| 競合 | 強み | 当社の差別化 |
|------|------|------------|
| GA4 | 行動データの無料収集・可視化 | ML予測＋SHAP説明（「次に何が起きるか」と「なぜか」）|
| Looker/Tableau | 高度な可視化・BIダッシュボード | アクション提案（逆SHAP）。「見るだけ」から「動く」へ |
| DataRobot | 高精度AutoML | 日本語CSV即対応、EC特化テンプレート、1/10のコスト |
| b→dash/aimstar | CRM・LTV管理 | 需要予測・在庫最適化との横断分析。SHAP要因可視化 |
| Silver Egg（レコメンド） | リアルタイムレコメンド | 横断ML分析プラットフォーム（補完関係） |

**最大の差別化**: EC事業者が「データはあるが分析する人がいない」ペインに対して、CSVアップロードだけでLightGBM + SHAP + 逆SHAPまでワンストップ提供。GA4やLookerは「見る」ツールだが、当社は「次のアクションを教える」ツール。

## 5. 結論

現在の4 MVP は L1〜L2 の下限に位置し、L3（プレミアムパック ¥260,000/月）到達には以下の3点が急務:

1. **ec-dashboard へのML予測+SHAP追加**（最優先: L1スターター完成のボトルネック）
2. **全アプリへの逆SHAP What-Ifシミュレーター実装**（L3キラー機能・競合との唯一無二の差別化）
3. **共通データ基盤構築による横断分析の実現**（L3ロックイン基盤・チャーン率低減）

技術面では ec-demo の ACTION_TEMPLATES が事業計画の「逆SHAP」に最も近い実装であり、この設計思想を他3アプリへ横展開することが最短でL3到達する経路となる。
