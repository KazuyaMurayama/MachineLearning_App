## 1. 評価フレームワーク

### 1.1 評価の視点

**従来のセグメント評価（§12A〜F）** は「市場としての魅力度」を評価していた。本レポートでは視点を180度転換し、**「接点ゼロの一人法人が、現実的に契約を獲得できるか」**を評価する。

### 1.2 評価軸（4軸 × 25点 = 100点満点）

| 評価軸 | 配点 | 評価内容 |
|--------|------|---------|
| **A. リーチ可能性** | 25点 | 意思決定者に到達できるか（チャネル・接点・障壁） |
| **B. プロモーション効果** | 25点 | コンテンツマーケ・記事投稿でブランディングが効くか |
| **C. 契約獲得難易度** | 25点 | 無償→有料転換・価格交渉・意思決定の速さ |
| **D. 開発・納品実現性** | 25点 | 一人で開発→デモ→導入→サポートまで回せるか |

### 1.3 各軸のスコア基準

```

20〜25点: 非常に高い実現性（一人でも十分可能）

```

  営業・提案:    30〜40h

  サポート:      10〜20h

### 2.2 活用可能な武器

| 武器 | 内容 | 効果 |
|------|------|------|
| **Claude Code** | アプリ開発3h/本、記事生成30分/本 | 開発・コンテンツの超効率化 |
| **Streamlit** | デモアプリを即構築 | 「動くもの」を見せて信頼獲得 |
| **無償/割引PoC** | 最初の2〜3社は無料〜半額 | 実績ゼロの壁を突破 |
| **価格競争力** | 大手コンサル比1/3〜1/5 | 中小企業の予算に合致 |
| **パーソナライズ提案** | Claude Codeで企業別提案書を自動生成 | 大量アプローチを少人数で |
| **成果保証** | 効果が出なければ無料 | リスクゼロの訴求 |

| 紹介・口コミ | 低 | 不定 | △ 最初は不可能 |
| Web広告（リスティング） | 中 | 即効 | ○ 予算次第 |

## 3. 食品製造業の実現性評価

### 3.1 リーチ可能性（A: 14/25点）

| 項目 | スコア | 根拠 |
|------|--------|------|
| A1. 意思決定者の特定 | 3/5 | 工場長・品質管理部長だが、Webで特定しにくい |
| A2. オンライン接触 | 2/5 | 製造業はWebでの情報収集が少ない、対面文化 |
| A3. 業界コミュニティ | 3/5 | 食品工業会・HACCP関連団体はあるが参入ハードル高 |
| A4. 紹介・口コミ | 3/5 | 業界内の横つながりはあるが、最初の1社が最難関 |
| A5. 一人法人への障壁 | 3/5 | 製造業は「実績」重視、法人規模を気にする傾向 |

**ボトルネック**: 製造業は対面営業が主流。Webだけでの接点構築は難しく、展示会（FOOMA等）への参加が事実上必須だが、一人での出展は非現実的。問合せフォーム営業は反応率が極めて低い（0.5%以下）。

### 3.2 プロモーション効果（B: 13/25点）

| 項目 | スコア | 根拠 |
|------|--------|------|
| B1. SNS/Web情報収集度 | 2/5 | 製造業の現場担当者はSNSをあまり見ない |
| B2. 業界メディア投稿 | 3/5 | 食品新聞・食品工業等あるが、寄稿ハードル高 |
| B3. SEO集客 | 3/5 | 「食品 需要予測 AI」等は検索ボリューム小 |
| B4. セミナー効果 | 2/5 | オンラインセミナーへの参加率が低い業界 |
| B5. 事例記事の拡散 | 3/5 | 事例があれば効くが、最初の事例獲得が困難 |

**現実**: 「食品製造 AI DX」で検索する人は少数。ブログ記事を量産してもリーチが限定的。業界メディアへの寄稿や、食品衛生協会のセミナー登壇が効果的だが、実績ゼロでは依頼が来ない。

### 3.3 契約獲得難易度（C: 12/25点）

| 項目 | スコア | 根拠 |
|------|--------|------|
| C1. 意思決定の速さ | 2/5 | 稟議制、工場長→本社の多段階承認が必要 |
| C2. 無償PoCの受容性 | 3/5 | 「無料ならやってみてもいい」が、データ提供に慎重 |
| C3. 価格感度 | 2/5 | 製造業のIT予算は限定的、既存ベンダーとの関係優先 |
| C4. 成果の定量化 | 3/5 | 需要予測精度・ロス削減率で定量化可能 |
| C5. スイッチングコスト | 2/5 | 既存システム（ERP等）との統合が前提になりがち |

**最大の壁**: 意思決定プロセスが遅い。中小食品メーカーでも「社長の鶴の一声」がないと動かない。データ提供への抵抗が強く（レシピ・製法の機密性）、NDAだけでは不十分。

### 3.4 開発・納品実現性（D: 18/25点）

| 項目 | スコア | 根拠 |
|------|--------|------|
| D1. データ取得の容易さ | 3/5 | ERPやCSVからの取得は可能だが、データ品質に課題 |
| D2. 技術的難易度 | 4/5 | LightGBM回帰が需要予測・品質予測に直接適用可 |
| D3. サポート負荷 | 3/5 | 製造業は「手取り足取り」のサポートを求めがち |
| D4. スケーラビリティ | 4/5 | 業種特化テンプレートで横展開しやすい |
| D5. 成果発現期間 | 4/5 | 需要予測は1〜2ヶ月で精度を検証可能 |

## 4. 小売/ECの実現性評価

### 4.1 リーチ可能性（A: 20/25点）

| 項目 | スコア | 根拠 |
|------|--------|------|
| A1. 意思決定者の特定 | 4/5 | EC事業者はWeb上で特定しやすい（ショップURL公開） |
| A2. オンライン接触 | 5/5 | EC事業者はSNS・メール・DMで接触可能 |
| A3. 業界コミュニティ | 4/5 | EC関連のオンラインコミュニティ・Slack多数 |
| A4. 紹介・口コミ | 3/5 | EC事業者同士の横つながりはあるが限定的 |
| A5. 一人法人への障壁 | 4/5 | EC業界はスタートアップに慣れており、法人規模を気にしない |

**強み**: EC事業者はWebネイティブ。X（Twitter）、note、LinkedIn等で直接接触可能。Shopify/BASEの事業者コミュニティに参加すれば自然にリーチできる。

### 4.2 プロモーション効果（B: 21/25点）

| 項目 | スコア | 根拠 |
|------|--------|------|
| B1. SNS/Web情報収集度 | 5/5 | EC事業者は積極的にWeb/SNSで情報収集する |
| B2. 業界メディア投稿 | 4/5 | ECのミカタ、ECCLAB等への寄稿ハードル低い |
| B3. SEO集客 | 4/5 | 「EC 需要予測」「LTV分析」は検索ボリュームあり |
| B4. セミナー効果 | 4/5 | EC事業者はウェビナー参加率が高い |
| B5. 事例記事の拡散 | 4/5 | EC事業者はnote/X/はてブで情報を共有する文化 |

**最大の強み**: Claude Codeで「EC×AI」のブログ記事を週3本量産すれば、3ヶ月でSEO上位が狙える。EC事業者はデータリテラシーが高く、「LightGBMで需要予測精度が○%向上」のような技術記事が刺さる。

| C2. 無償PoCの受容性 | 4/5 | 「データ渡すだけならやってみたい」の反応が得やすい |
| C3. 価格感度 | 3/5 | EC事業者はコスト意識が高く、ROI重視 |
| C4. 成果の定量化 | 4/5 | 売上予測精度・LTV・解約率で明確に定量化 |
| C5. スイッチングコスト | 3/5 | 既存のGA4/Shopify分析との差別化が必要 |

**注意点**: EC事業者はツールを試すのも早いが、辞めるのも早い。「効果が出なければすぐ解約」の文化。成果保証が効果的だが、解約率も高くなりがち。

### 4.4 開発・納品実現性（D: 21/25点）

| D2. 技術的難易度 | 4/5 | LightGBMが売上予測・LTV予測に最適 |
| D3. サポート負荷 | 4/5 | EC事業者は自分でダッシュボードを使える |

## 5. 建設業の実現性評価

### 5.1 リーチ可能性（A: 11/25点）

| 項目 | スコア | 根拠 |
|------|--------|------|
| A1. 意思決定者の特定 | 2/5 | 現場所長・経営者だが、Webでの特定が困難 |
| A2. オンライン接触 | 2/5 | 建設業は電話・FAX文化、メール営業の反応率極低 |
| A3. 業界コミュニティ | 2/5 | 建設業のオンラインコミュニティは少ない |
| A4. 紹介・口コミ | 3/5 | 元請-下請の縦のつながりはあるが、横は限定的 |
| A5. 一人法人への障壁 | 2/5 | 建設業は「会社の規模」「実績」を最重視 |

**ボトルネック**: 建設業は最もオフライン志向の業界。Web経由でのリーチはほぼ不可能。建設DX展等の展示会への参加が事実上必須だが、一人での出展は困難。ANDPAD等の既存ベンダーの代理店/パートナーになるルートも検討すべき。

### 5.2 プロモーション効果（B: 10/25点）

| 項目 | スコア | 根拠 |
|------|--------|------|
| B1. SNS/Web情報収集度 | 1/5 | 現場監督・経営者がSNSで情報収集する率は極めて低い |
| B2. 業界メディア投稿 | 2/5 | 日経クロステック建設等あるが寄稿ハードル高い |
| B3. SEO集客 | 2/5 | 「建設 AI」の検索者はごく少数 |
| B4. セミナー効果 | 2/5 | 建設業者のウェビナー参加率は低い |
| B5. 事例記事の拡散 | 3/5 | 事例があれば業界内で口コミが広がりやすい |

**現実**: Claude Codeでブログを量産しても、建設業の意思決定者には届かない。SNS/SEOの効果が最も低いセグメント。

### 5.3 契約獲得難易度（C: 10/25点）

## 17. 結論・最終推奨

### 17.1 一人法人にとっての最適戦略

```
最終推奨
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【第1優先】小売/EC（80点）
  → Webリーチ最容易、ブログ量産が最も効く
  → Shopify/BASE APIで開発効率も最高
  → 一人法人との親和性No.1

【第2優先】士業（79点）
  → リーチ手段が豊富（税理士会・freeeパートナー等）
  → 「AI不安」がコンテンツ需要を生む
  → 士業自身が少人数事務所で一人法人への障壁最低

【条件付き推奨】美容（67点）・歯科（65点）
  → Tier1で実績を積んでから参入
  → 価格調整が必須（美容は¥3〜5万/月に）

【非推奨】食品製造（57点）
  → 紹介案件のみ受動的に対応

【参入不可】建設（45点）
  → 一人法人では成立しない
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 17.2 §12A〜Fの評価との差分

| セグメント | §12A〜Fでの評価 | 本レポートの評価 | 差分の理由 |
|-----------|----------------|----------------|-----------|
| 小売/EC | ○ 高評価 | **◎ 最高評価** | 一人法人との親和性が最も高い |
| 士業 | ◎ 最高評価 | **◎ 最高評価** | 変わらず。ただし2番手に |
| 美容 | ○ 中〜高評価 | **○ 条件付き** | 価格感度の壁を過小評価していた |
| 歯科 | ○ 中〜高評価 | **○ 条件付き** | セミナー経由のリーチが前提 |
| 食品製造 | △ 中評価 | **△ 非推奨寄り** | 対面文化がWebマーケを無効化 |
| 建設 | △ 中評価 | **× 参入不可** | 一人法人ではリーチ自体が不成立 |
