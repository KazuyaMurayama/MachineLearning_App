## 1. 評価フレームワーク

### 1.1 評価の視点

**従来のセグメント評価（§12A〜F）** は「市場としての魅力度」を評価していた。本レポートでは視点を180度転換し、**「接点ゼロの一人法人が、現実的に契約を獲得できるか」**を評価する。

### 1.2 評価軸（5軸 × 20点 = 100点満点）

> **v1（旧版）からの変更点**: 旧版は4軸×25点で「リーチ〜納品」までの実現性のみを評価していた。しかし「契約後にいくら稼げるか」「競合にどう勝つか」が欠落していたため、**E. 収益性・LTV**軸を追加し、5軸×20点に再構成した。

| 評価軸 | 配点 | 評価内容 |
|--------|------|---------|
| **A. リーチ可能性** | 20点 | 意思決定者に到達できるか（チャネル・接点・障壁） |
| **B. プロモーション効果** | 20点 | コンテンツマーケ・記事投稿でブランディングが効くか |
| **C. 契約獲得難易度** | 20点 | 無償→有料転換・価格交渉・意思決定の速さ・競合環境 |
| **D. 開発・納品実現性** | 20点 | 一人で開発→デモ→導入→サポートまで回せるか |
| **E. 収益性・LTV** | 20点 | 単価・継続率・解約リスク・価格優位性・収益の持続可能性 |

## 3. 食品製造業の実現性評価

### 3.1 リーチ可能性（A: 11/20点）

| 項目 | スコア | 根拠 |
|------|--------|------|
| A1. 意思決定者の特定 | 2/4 | 工場長・品質管理部長だが、Webで特定しにくい |
| A2. オンライン接触 | 2/4 | 製造業はWebでの情報収集が少ない、対面文化 |
| A3. 業界コミュニティ | 2/4 | 食品工業会・HACCP関連団体はあるが参入ハードル高 |
| A4. 紹介・口コミ | 3/4 | 業界内の横つながりはあるが、最初の1社が最難関 |
| A5. 一人法人への障壁 | 2/4 | 製造業は「実績」重視、法人規模を気にする傾向 |

**ボトルネック**: 製造業は対面営業が主流。Webだけでの接点構築は難しく、展示会（FOOMA等）への参加が事実上必須だが、一人での出展は非現実的。問合せフォーム営業は反応率が極めて低い（0.5%以下）。

### 3.2 プロモーション効果（B: 10/20点）

| 項目 | スコア | 根拠 |
|------|--------|------|
| B1. SNS/Web情報収集度 | 1/4 | 製造業の現場担当者はSNSをあまり見ない |
| B2. 業界メディア投稿 | 2/4 | 食品新聞・食品工業等あるが、寄稿ハードル高 |
| B3. SEO集客 | 3/4 | 「食品 需要予測 AI」等は検索ボリューム小 |
| B4. セミナー効果 | 2/4 | オンラインセミナーへの参加率が低い業界 |
| B5. 事例記事の拡散 | 2/4 | 事例があれば効くが、最初の事例獲得が困難 |

**現実**: 「食品製造 AI DX」で検索する人は少数。ブログ記事を量産してもリーチが限定的。業界メディアへの寄稿や、食品衛生協会のセミナー登壇が効果的だが、実績ゼロでは依頼が来ない。

### 3.3 契約獲得難易度（C: 10/20点）

| 項目 | スコア | 根拠 |
|------|--------|------|
| C1. 意思決定の速さ | 2/4 | 稟議制、工場長→本社の多段階承認が必要 |
| C2. 無償PoCの受容性 | 2/4 | 「無料ならやってみてもいい」が、データ提供に慎重 |
| C3. 価格感度 | 2/4 | 製造業のIT予算は限定的、既存ベンダーとの関係優先 |
| C4. 成果の定量化 | 2/4 | 需要予測精度・ロス削減率で定量化可能 |
| C5. スイッチングコスト | 2/4 | 既存システム（ERP等）との統合が前提になりがち |

**最大の壁**: 意思決定プロセスが遅い。中小食品メーカーでも「社長の鶴の一声」がないと動かない。データ提供への抵抗が強く（レシピ・製法の機密性）、NDAだけでは不十分。

### 3.4 開発・納品実現性（D: 14/20点）

| 項目 | スコア | 根拠 |
|------|--------|------|
| D1. データ取得の容易さ | 2/4 | ERPやCSVからの取得は可能だが、データ品質に課題 |
| D2. 技術的難易度 | 3/4 | LightGBM回帰が需要予測・品質予測に直接適用可 |
| D3. サポート負荷 | 3/4 | 製造業は「手取り足取り」のサポートを求めがち |
| D4. スケーラビリティ | 3/4 | 業種特化テンプレートで横展開しやすい |
| D5. 成果発現期間 | 3/4 | 需要予測は1〜2ヶ月で精度を検証可能 |

### 3.5 収益性・LTV（E: 12/20点）← 新設

| 項目 | スコア | 根拠 |
|------|--------|------|
| E1. 実現可能な月額単価 | 3/4 | ¥10〜20万/月は製造業の感覚では許容範囲 |
| E2. 契約継続率 | 3/4 | 一度導入すれば継続性は高い（製造ラインに組み込まれる） |
| E3. 大手競合への価格優位性 | 3/4 | 大手SIer比1/5〜1/10で明確な価格優位性あり |
| E4. 無料/低価格ツールとの差別化 | 2/4 | 汎用BIツール（Power BI等）との差別化が必要 |
| E5. LTVの大きさ | 1/4 | 契約までの営業コストが高く、回収に時間がかかる |

**収益性の評価**: 単価・継続率は良いが、契約獲得までのコスト（時間・対面営業）が重く、ROI回収期間が長い。

## 4. 小売/ECの実現性評価

### 4.1 リーチ可能性（A: 16/20点）

| 項目 | スコア | 根拠 |
|------|--------|------|
| A1. 意思決定者の特定 | 3/4 | EC事業者はWeb上で特定しやすい（ショップURL公開） |
| A2. オンライン接触 | 4/4 | EC事業者はSNS・メール・DMで接触可能 |
| A3. 業界コミュニティ | 3/4 | EC関連のオンラインコミュニティ・Slack多数 |
| A4. 紹介・口コミ | 3/4 | EC事業者同士の横つながりはあるが限定的 |
| A5. 一人法人への障壁 | 3/4 | EC業界はスタートアップに慣れており、法人規模を気にしない |

**強み**: EC事業者はWebネイティブ。X（Twitter）、note、LinkedIn等で直接接触可能。Shopify/BASEの事業者コミュニティに参加すれば自然にリーチできる。

### 4.2 プロモーション効果（B: 17/20点）

| 項目 | スコア | 根拠 |
|------|--------|------|
| B1. SNS/Web情報収集度 | 4/4 | EC事業者は積極的にWeb/SNSで情報収集する |
| B2. 業界メディア投稿 | 3/4 | ECのミカタ、ECCLAB等への寄稿ハードル低い |
| B3. SEO集客 | 3/4 | 「EC 需要予測」「LTV分析」は検索ボリュームあり |
| B4. セミナー効果 | 3/4 | EC事業者はウェビナー参加率が高い |
| B5. 事例記事の拡散 | 4/4 | EC事業者はnote/X/はてブで情報を共有する文化 |

**最大の強み**: Claude Codeで「EC×AI」のブログ記事を週3本量産すれば、3ヶ月でSEO上位が狙える。EC事業者はデータリテラシーが高く、「LightGBMで需要予測精度が○%向上」のような技術記事が刺さる。

### 4.3 契約獲得難易度（C: 14/20点）

| 項目 | スコア | 根拠 |
|------|--------|------|
| C1. 意思決定の速さ | 4/4 | 代表やEC担当が即決できるケースが多い |
| C2. 無償PoCの受容性 | 3/4 | 「データ渡すだけならやってみたい」の反応が得やすい |
| C3. 価格感度 | 2/4 | EC事業者はコスト意識が高く、ROI重視 |
| C4. 成果の定量化 | 3/4 | 売上予測精度・LTV・解約率で明確に定量化 |
| C5. スイッチングコスト | 2/4 | 既存のGA4/Shopify分析との差別化が必要 |

**注意点**: EC事業者はツールを試すのも早いが、辞めるのも早い。「効果が出なければすぐ解約」の文化。成果保証が効果的だが、解約率も高くなりがち。

### 4.4 開発・納品実現性（D: 17/20点）

| 項目 | スコア | 根拠 |
|------|--------|------|
| D1. データ取得の容易さ | 4/4 | Shopify/BASE/GA4のAPIでデータ取得が容易 |
| D2. 技術的難易度 | 3/4 | LightGBMが売上予測・LTV予測に最適 |
| D3. サポート負荷 | 3/4 | EC事業者は自分でダッシュボードを使える |
| D4. スケーラビリティ | 4/4 | Shopifyプラグイン化等でスケール可能 |
| D5. 成果発現期間 | 3/4 | 売上予測は1ヶ月で精度検証可能 |

### 4.5 収益性・LTV（E: 11/20点）← 新設

| 項目 | スコア | 根拠 |
|------|--------|------|
| E1. 実現可能な月額単価 | 2/4 | ¥5〜10万/月。EC事業者のコスト意識は高く、上限がある |
| E2. 契約継続率 | 1/4 | **解約リスクが最も高い**。「効果が出なければすぐ解約」の文化。月次解約率10〜15%も想定 |
| E3. 大手競合への価格優位性 | 3/4 | DATAFLUCT等の大手比1/3〜1/5で明確な価格優位性 |
| E4. 無料/低価格ツールとの差別化 | 2/4 | **GA4・Shopify Analytics・Looker Studio等の無料ツールが強力**。「無料で8割できる」状態 |
| E5. LTVの大きさ | 3/4 | 単価×継続率で算出すると、解約リスクの高さがLTVを圧縮 |

**収益性の評価**: リーチ・プロモは最強だが、**解約率の高さと無料ツールとの競合**が収益性を大きく押し下げる。「試すのも早いが辞めるのも早い」業界特性がLTVに直結。

## 17. 結論・最終推奨

### 17.1 一人法人にとっての最適戦略

> **v2での変更**: 収益性・LTV軸を追加した結果、**士業が総合1位に浮上**。小売/ECは解約率の高さと無料ツール競合でLTVが圧縮され2位に。

```
最終推奨（v2: 収益性・LTV反映版）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【第1優先】士業（80点）★ v1から順位上昇
  → リーチ手段が豊富（税理士会・freeeパートナー等）
  → 「AI不安」がコンテンツ需要を生む
  → 士業自身が少人数事務所で一人法人への障壁最低
  → 船井総研比1/4〜1/6の価格優位性が明確
  → 顧問契約文化で継続率が最も高く、LTV最大

【第2優先】小売/EC（75点）★ v1から順位下降
  → Webリーチ最容易、ブログ量産が最も効く
  → Shopify/BASE APIで開発効率最高
  → ⚠ 解約率が高く、GA4等の無料ツールとの競合あり
  → ⚠ LTVは士業に大きく劣る

【条件付き推奨】歯科（65点）> 美容（61点）★ 順位逆転
  → 歯科は美容より価格許容度・継続率で優位
  → 美容は既存SaASと同価格帯で薄利多売リスク
  → Tier1で実績を積んでから参入

【非推奨】食品製造（57点）
  → 紹介案件のみ受動的に対応

【参入不可】建設（40点）★ v1から更に下降
  → 一人法人では成立しない。既存SaaSより割高
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```
