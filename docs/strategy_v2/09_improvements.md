## Part 4: クロスチェック体制

### 4.1 チェック方法の設計

§8の数値を3つの独立した方法で検証する。

| チェック方法 | 目的 | 検証対象 |
|------------|------|---------|
| **① 時間予算積み上げ** | 全活動が月32hに収まるか | 各Phase×活動項目の合計 |
| **② 売上整合性検証** | MRR = 顧客数 × ARPU、二重計上なし | 月次テーブルの数学的整合 |
| **③ チャーン数学検証** | 解約が正しく織り込まれているか | 純増 = 新規 - (既存×チャーン率) |
| **④ ベンチマーク比較** | 業界水準との乖離チェック | ARPU、チャーン、営業サイクル |
| **⑤ 感度分析** | 前提が崩れた場合のインパクト | 4つの主要変数を変動させる |

### 4.2 チェック① 時間予算の検証結果

| Phase | 月間合計 | 判定 | 備考 |
|-------|---------|------|------|
| M1-M3（0社） | 22-31h | ✅ 余裕 | 開発中心。問題なし |

### 4.4 チェック③ チャーン数学の検証結果

月次チャーン3%の場合：

| 期間 | 期首平均顧客 | 月間期待解約 | 期間合計解約 |
|------|------------|------------|------------|
| Y1 M4-M12 | 3.5社 | 0.11社 | **0.96社** |
| Y2 M13-M24 | 10社 | 0.30社 | **3.60社** |
| **合計** | | | **4.56社** |

**必要な新規獲得:**
- M24目標12社 = 初期0社 + 新規獲得 - 解約
- 新規獲得 = 12 + 4.56 = **16.56社**（24ヶ月で約17社の新規が必要）

**⚠️ 発見:** §8の標準シナリオでは解約が明示的に計上されていない箇所がある。
ただし、ARPU向上で解約の売上インパクトを相殺する設計になっている可能性あり。

**修正:** 解約を明示的に織り込み、新規獲得数を17社に修正。
月平均0.7社の新規獲得（M4以降の21ヶ月）= 約3週間に1社のペース。
