## 4エージェントによる分析の統合

§7のソロ実行プランを、4つの専門エージェントが多角的に検証・改善した。

| エージェント | 主な提案 | Devil's Advocate の判定 |
|------------|---------|----------------------|
| 価格戦略 | ARPU ¥68K→¥148K（+118%） | **過剰** → ¥80-100Kが現実的上限 |
| 時間レバレッジ | 顧客対応 4.0h→0.25h/社 | **過大評価** → 0.8-1.5h/社が現実的 |
| 収益多角化 | 3つの追加収益源で+¥935万 | **過負荷** → 1つに絞るべき |
| Devil's Advocate | 3提案の合計は月40-47hで破綻 | 月32h以内に収める修正案を提示 |

### 核心の発見：時間の壁

```
3エージェントの提案を合算した月間稼働:

  M4-M6:   37時間（上限32hを+16%超過）
  M7-M12:  39-44時間（上限32hを+22-38%超過）
  M13-M24: 39-47時間（上限32hを+22-47%超過）

→ 何かを「やらない」と決めなければ、計画は月32hに収まらない
```

## 統合版：修正後の戦略

### 価格戦略（修正版）

エージェント1の「現在価格は安すぎる」という診断は正しい。
ただし¥150K/月は中小製造業の即決閾値を超える。

**修正版価格テーブル:**

| プラン | 月額 | 初期費用 | ターゲット | 提供開始 |
|--------|------|---------|-----------|---------|
| **Light** | ¥50,000 | ¥200,000 | 20-50名規模。初めてAIを使う企業 | M1 |
| **Standard** | ¥100,000 | ¥400,000 | 50-150名規模。見積の属人化が深刻 | M1 |
| **Professional** | ¥150,000 | ¥600,000 | 100名超。補助金利用前提 | M7 |
| ~~Enterprise~~ | ~~削除~~ | - | 1人運営でEnterprise対応は不可能 | - |

**判断の根拠:**
- 月¥100Kは板金加工の社長が「社長決裁」で即決できるギリギリの金額
- 月¥150Kは稟議書が必要 → 営業サイクルが2-3倍に伸びる → 補助金利用者限定
- Enterpriseは「24時間対応」「カスタム」「SLA」を要求される → 他の全顧客に悪影響

**想定ARPU推移:**

```
M1-M6:   ¥65,000（Light多め）
M7-M12:  ¥80,000（Standard中心に移行）
M13-M18: ¥90,000（Professional少数追加）
M19-M24: ¥100,000（値上げ + 上位プラン比率増）
```

### 時間設計（修正版）

エージェント2の自動化パイプラインの設計は優秀だが、効果の見積もりが楽観的。

**修正後の顧客対応時間:**

| 時期 | 顧客対応/社/月 | 内訳 |
|------|--------------|------|
| M1-M6 | **1.5h** | 電話15分 + レポート確認20分 + 問い合わせ対応30分 + 事務15分 |
| M7-M12 | **1.0h** | 電話15分 + 自動レポート確認10分 + 問い合わせ10分 + 事務15分 |
| M13-M18 | **0.8h** | 電話15分 + 自動レポート確認5分 + FAQ化された問い合わせ5分 + 事務10分 |
| M19-M24 | **0.8h** | 安定。これ以下は製造業相手では非現実的 |

**重要:** 「月1回15分の電話フォロー」は削れない。製造業顧客は「人間との接点」がないと解約する。自動レポートは「電話のアジェンダ」として使う。

**月間稼働の上限管理:**

```
月32時間の配分（M12以降の安定期）:

  顧客対応（10社×0.8h）     8.0h   25%
  開発・改善                  4.0h   13%
  営業・パートナー            5.0h   16%
  スポットコンサル（月1件）   5.0h   16%
  自動化・仕組み化            4.0h   13%
  管理業務（経理・契約）      3.0h    9%
  マーケティング              3.0h    9%
  ─────────────────────────────────
  合計                       32.0h  100%  ← 上限ちょうど
```

### 収益多角化（修正版）

3つ → 1つに絞る。

| 収益源 | 判定 | 理由 |
|--------|------|------|
| **スポットコンサル** | **採用（M7以降）** | 時間単価が最高。既存顧客からの依頼限定で営業不要 |
| ~~ウェビナー~~ | **却下** | 板金加工の経営者（50-70代）はウェビナーに参加しない。集客コストが時間制約と合わない |
| ~~IT導入補助金パートナー~~ | **後回し（M18以降）** | 申請業務は行政書士/診断士と提携し、紹介手数料モデルに |

**スポットコンサルの設計:**

```
■ 「見積改善診断」パッケージ

  対象: 既存顧客 or 既存顧客からの紹介のみ（新規営業はしない）
  価格: ¥150,000-250,000/回
  内容: 見積データ分析 → 利益率の低い案件パターン特定 → 改善提案
  所要時間: 5時間（データ受領1h + Claude Code分析2h + レポート作成1h + 報告1h）
  頻度: 月1件（最大2件。月の稼働が28hを超えたら翌月は受けない）

  副次効果: 「このAIツールを入れれば自動でできますよ」→ 導入のアップセル導線
```

### 自動化投資（修正版）

エージェント2の自動化設計は採用するが、投資タイミングを後ろ倒し。

| 時期 | 自動化投資 | 理由 |
|------|-----------|------|
| M1-M6 | **しない**（手動運用） | 顧客0-5社で自動化は過剰投資。手動で顧客理解を深める |
| M7-M9 | **月次レポート自動化**（8h投資） | 顧客5社超で効果が出始める |
| M10-M12 | **データ前処理テンプレート化**（8h投資） | 新規顧客のオンボーディング効率化 |
| M13-M18 | **FAQ・問い合わせ半自動化**（8h投資） | 定型質問が蓄積された段階で実施 |
| M19-M24 | **モデル再学習の自動化**（8h投資） | 顧客10社超で手動再学習が限界に |

合計投資: 32時間（元提案の90時間から大幅削減）
