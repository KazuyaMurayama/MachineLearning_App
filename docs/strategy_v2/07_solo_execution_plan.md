## 前提の根本的転換

これまでのレポート（§1-6）は **チーム5-25名・資金調達あり** を前提としていた。
本セクションでは、以下の現実に基づき **すべてを再設計** する。

| 制約 | 内容 |
|------|------|
| 稼働者 | **1名（あなたのみ）** |
| 月間稼働時間 | **16-32時間**（1日30-90分） |
| 年間稼働時間 | **192-384時間** |
| レバレッジ | **Claude Code / Claude Co-work をフル活用** |
| 専門性 | データサイエンス（ML）、Claude Code、マーケティング |
| 資金調達 | **なし（ブートストラップ）** |

### この制約が意味すること

```
年間384時間（最大） ÷ 時給換算¥10,000 = ¥3,840,000分の労働力

比較:
  フルタイム1名 = 年間2,000時間 = あなたの5-10倍
  前レポートのY1チーム5名 = あなたの25-50倍

→ あなたの384時間を「どこに使うか」が戦略のすべて
```

### Claude Codeの力を正確に見積もる

Claude Code は万能ではない。何ができて何ができないかを正確に理解する：

| Claude Codeが得意 | Claude Codeが苦手 |
|---|---|
| コード実装・リファクタリング | 顧客との関係構築 |
| テスト作成・デバッグ | 業界特有のドメイン知識の深堀り |
| ドキュメント生成 | 営業・クロージング |
| データ分析スクリプト作成 | 顧客のデータ整備の伴走 |
| UI/UXの実装 | 信頼の醸成（「人」が必要） |
| API設計・統合 | 展示会・セミナーでの登壇 |

**あなたの時間配分の原則:**
- **Claude Codeに任せられること → 指示を出して待つ（実質5-10分）**
- **あなたにしかできないこと → 残りの時間をすべて投入**

## 戦略の再設計: ソロ×少時間に最適化

### 選択した戦略: 「専門家プロダクタイズドサービス」

前レポートの選択肢の中から、ソロ×少時間に最適なモデルを選定：

| 却下した選択肢 | 却下理由 |
|---|---|
| ~~SaaS（セルフサービス）~~ | カスタマーサポート対応が不可能。1人で24時間対応は無理 |
| ~~コンサル+ツール（90日伴走型）~~ | 5社同時伴走 = 月80時間以上。稼働時間を超過 |
| ~~3業界同時展開~~ | 1人で3業界のドメイン知識を深めるのは不可能 |

**選択した戦略:**

```
┌──────────────────────────────────────────────────────────┐
│                                                          │
│   「製造業（板金加工）特化の見積AI」                      │
│    × プロダクタイズドサービス                             │
│    × 非同期コンサル                                       │
│                                                          │
│   ┌──────────────────────────────────────────────────┐   │
│   │ 提供形態:                                        │   │
│   │   ① ツール（Streamlit App）: セルフサービス      │   │
│   │   ② 月1回の分析レポート: Claude Code で自動生成  │   │
│   │   ③ 非同期Q&A: メール/Slack（1日1回まとめ返信） │   │
│   │                                                  │   │
│   │ 価格:                                            │   │
│   │   月額 ¥50,000-100,000（企業規模による）         │   │
│   │   初期導入費 ¥200,000-500,000（データ整備込み）  │   │
│   └──────────────────────────────────────────────────┘   │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

### なぜ板金加工1本に絞るのか

| 理由 | 詳細 |
|------|------|
| **1. 市場サイズが「ちょうどいい」** | 約3,000社。大手SIerが参入するには小さすぎ、個人が攻めるには十分 |
| **2. 課題が明確で深刻** | ベテラン退職 → 見積ができなくなる → 経営危機。「Nice to have」ではなく「Must have」 |
| **3. データが扱いやすい** | 材料費、加工時間、数量 → 価格。構造化データが既にある企業が多い |
| **4. あなたの時間で回せる** | 企業ごとのカスタマイズが少ない（見積構造が業界で類似） |
| **5. 口コミが起きやすい** | 業界団体、協同組合、展示会コミュニティが密。1社の成功が5社を呼ぶ |
| **6. 補助金が使える** | IT導入補助金（最大75%補助）、ものづくり補助金 |

