## 前提の根本的転換

これまでのレポート（§1-6）は **チーム5-25名・資金調達あり** を前提としていた。
本セクションでは、以下の現実に基づき **すべてを再設計** する。

| 制約 | 内容 |
|------|------|
| 稼働者 | **1名（あなたのみ）** |
| 月間稼働時間 | **16-32時間**（1日30-90分） |
| 年間稼働時間 | **192-384時間** |
| レバレッジ | **Claude Code / Claude Co-work をフル活用** |
| 専門性 | データサイエンス（ML）、Claude Code、マーケティング |
| 資金調達 | **なし（ブートストラップ）** |

### この制約が意味すること

```
年間384時間（最大） ÷ 時給換算¥10,000 = ¥3,840,000分の労働力

比較:
  フルタイム1名 = 年間2,000時間 = あなたの5-10倍
  前レポートのY1チーム5名 = あなたの25-50倍

→ あなたの384時間を「どこに使うか」が戦略のすべて
```

### Claude Codeの力を正確に見積もる

Claude Code は万能ではない。何ができて何ができないかを正確に理解する：

| Claude Codeが得意 | Claude Codeが苦手 |
|---|---|
| コード実装・リファクタリング | 顧客との関係構築 |
| テスト作成・デバッグ | 業界特有のドメイン知識の深堀り |
| ドキュメント生成 | 営業・クロージング |
| データ分析スクリプト作成 | 顧客のデータ整備の伴走 |
| UI/UXの実装 | 信頼の醸成（「人」が必要） |
| API設計・統合 | 展示会・セミナーでの登壇 |

**あなたの時間配分の原則:**
- **Claude Codeに任せられること → 指示を出して待つ（実質5-10分）**
- **あなたにしかできないこと → 残りの時間をすべて投入**

## 戦略の再設計: ソロ×少時間に最適化

### 選択した戦略: 「専門家プロダクタイズドサービス」

前レポートの選択肢の中から、ソロ×少時間に最適なモデルを選定：

| 却下した選択肢 | 却下理由 |
|---|---|
| ~~SaaS（セルフサービス）~~ | カスタマーサポート対応が不可能。1人で24時間対応は無理 |
| ~~コンサル+ツール（90日伴走型）~~ | 5社同時伴走 = 月80時間以上。稼働時間を超過 |
| ~~3業界同時展開~~ | 1人で3業界のドメイン知識を深めるのは不可能 |

**選択した戦略:**

```
┌──────────────────────────────────────────────────────────┐
│                                                          │
│   「製造業（板金加工）特化の見積AI」                      │
│    × プロダクタイズドサービス                             │
│    × 非同期コンサル                                       │
│                                                          │
│   ┌──────────────────────────────────────────────────┐   │
│   │ 提供形態:                                        │   │
│   │   ① ツール（Streamlit App）: セルフサービス      │   │
│   │   ② 月1回の分析レポート: Claude Code で自動生成  │   │
│   │   ③ 非同期Q&A: メール/Slack（1日1回まとめ返信） │   │
│   │                                                  │   │
│   │ 価格:                                            │   │
│   │   月額 ¥50,000-100,000（企業規模による）         │   │
│   │   初期導入費 ¥200,000-500,000（データ整備込み）  │   │
│   └──────────────────────────────────────────────────┘   │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

### なぜ板金加工1本に絞るのか

| 理由 | 詳細 |
|------|------|
| **1. 市場サイズが「ちょうどいい」** | 約3,000社。大手SIerが参入するには小さすぎ、個人が攻めるには十分 |
| **2. 課題が明確で深刻** | ベテラン退職 → 見積ができなくなる → 経営危機。「Nice to have」ではなく「Must have」 |
| **3. データが扱いやすい** | 材料費、加工時間、数量 → 価格。構造化データが既にある企業が多い |
| **4. あなたの時間で回せる** | 企業ごとのカスタマイズが少ない（見積構造が業界で類似） |
| **5. 口コミが起きやすい** | 業界団体、協同組合、展示会コミュニティが密。1社の成功が5社を呼ぶ |
| **6. 補助金が使える** | IT導入補助金（最大75%補助）、ものづくり補助金 |

## 月別アクションプラン（12ヶ月）

### あなたの時間の使い方の原則

```
■ 1日のルーティン（30-90分）

朝（指示フェーズ: 10-20分）:
  → Claude Code / Co-work に開発タスクを指示
  → 顧客からのメール/Slackを確認、返信をドラフト

日中（待機フェーズ: 0分）:
  → Claude が作業中。あなたは本業に集中

夜（確認フェーズ: 20-70分）:
  → Claude の成果物を確認・フィードバック
  → 顧客対応の返信を送信
  → マーケティングコンテンツの確認・投稿
  → 翌日の計画
```

### 最も重要な原則: 「やらないことリスト」

限られた時間で成果を出すために、**やらないこと**を明確にする：

| やらないこと | 理由 |
|---|---|
| ❌ 3業界同時展開 | 1業界すら十分にできない |
| ❌ 完璧なUI/UX | 動けば十分。見た目より価値 |
| ❌ 自前のインフラ構築 | Streamlit Cloud or Railway で十分 |
| ❌ ブログの毎週更新 | 月1本の事例記事で十分 |
| ❌ SNS運用の頻繁な投稿 | 週1回のLinkedIn投稿で十分 |
| ❌ 展示会出展 | コスト高すぎ。オンラインセミナーで代替 |
| ❌ カスタム開発の請負 | 時間を食い潰す。標準プロダクトのみ |
| ❌ 価格交渉への応じすぎ | 補助金を案内して正規価格を維持 |
| ❌ Next.js等への技術移行 | Streamlitのまま Year 1 は乗り切る |
| ❌ 自前の決済システム | Stripe or 請求書払い（手動）で十分 |

## リスクと対策

| リスク | 確率 | 影響 | 対策 |

| 競合（大手SIer）参入 | 15% | 中 | 板金加工に特化することで差別化。大手は汎用ツール止まり |
| Claude Code APIコスト増 | 20% | 低 | 顧客数に応じて価格転嫁可能 |

## 最後に: あなたへのメッセージ

月16-32時間で年間¥250万（最もありそうケース）は、**時間あたりの生産性としては非常に高い**。フルタイムのデータサイエンティストの年収¥600-1,000万と比較しても、時間あたりでは同等以上。

成功の鍵は **「やること」ではなく「やらないこと」を守ること**。

```
✅ やるべきこと
  1. Claude Code で開発を高速化（あなたの時間を10倍レバレッジ）
  2. 板金加工1業界に100%集中
  3. 無償PoCで信頼を勝ち取り、紹介の連鎖を作る
  4. 「見積の精度」より「見積の根拠を可視化」で売る
  5. 補助金を活用して顧客の費用障壁を下げる

❌ やってはいけないこと
  1. 3業界同時に手を出す
  2. 完璧なプロダクトを作ろうとする
  3. 価格を下げて顧客数を増やそうとする
  4. 技術スタックの移行に時間を使う
  5. 月32時間を超えて稼働する（燃え尽きる）
```
