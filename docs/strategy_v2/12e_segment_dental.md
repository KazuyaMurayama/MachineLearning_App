## 1. エグゼクティブサマリー

### 歯科クリニック×ML予測アプリの機会

**市場規模**: AI in Dentistry市場は$459.6M（2025年）→ $3,263.9M（2034年）、CAGR 21.78%で急成長中。日本の歯科診療所数は約67,000施設（コンビニ超え）で競争激化が深刻。歯科経営管理ソフト市場も$2.47B（2025年）→ $5.33B（2035年）とCAGR 7.2%で拡大。

**セグメント評価スコア: 3.8 / 5.0**（Phase 2推奨）

| 評価項目 | スコア | 根拠 |
|---------|--------|------|
| 市場規模・成長性 | ★★★★☆ | 6.7万施設、AI歯科CAGR 21.78% |
| ペインの深さ | ★★★★★ | 患者中断・自費率が経営直結 |
| 技術適合性 | ★★★★☆ | LightGBM回帰が来院予測・自費予測に直接適用可 |
| 競合状況 | ★★★★☆ | レセコン・予約は飽和、ML分析は空白地帯 |
| 導入障壁 | ★★★☆☆ | 院長の意思決定が速い反面、ITリテラシーに差 |
| 単価ポテンシャル | ★★★★☆ | 医療法人は月額10〜20万円の投資余力あり |

**核心的インサイト**:
- 歯科医院の最大ペインは**治療中断・患者離脱**（中断率20〜30%、年間損失数百万円）
- 既存システム（レセコン・予約管理）は業務効率化が主で**予測分析機能が皆無**
- AI活用事例：ノーショー予測による50%削減、AI診断補助で症例受入率10〜20%向上
- 歯科医院の20%がAI予測分析をクラウドで新規導入（2025年時点）

**推奨戦略**: 医療法人・分院展開クリニック（3院以上）をメインターゲットに、患者離脱予測AI + 売上予測をコアとしたスイート提供。美容業界（§12D）と類似のペイン構造を活かし、開発資産を共有しつつ歯科特化UI/UXでローカライズする。

## 3. ターゲット企業群の定義

### 3.1 セグメンテーション

| セグメント | 院数 | 年商規模 | 推定数 | 優先度 |
|-----------|------|---------|--------|--------|
| **Tier 1: 大手医療法人** | 10院以上 | 5億円以上 | 約200法人 | ★★★★★ |
| **Tier 2: 中規模法人** | 3〜9院 | 1〜5億円 | 約800法人 | ★★★★★ |
| **Tier 3: 小規模法人・複数院** | 2院 | 5,000万〜1億円 | 約2,000法人 | ★★★★☆ |
| **Tier 4: 個人開業医** | 1院 | 〜5,000万円 | 約60,000院 | ★★☆☆☆ |

### 3.2 メインターゲット：Tier 1-2（医療法人・分院展開）

**選定理由**:
- **データ量**: レセコン・電子カルテから蓄積された患者データが豊富
- **投資余力**: 月額10〜20万円のIT投資が可能な財務基盤
- **経営課題の顕在化**: 分院間の業績差、スタッフ配置、自費率格差が経営課題
- **意思決定者**: 理事長・院長が経営的視点を持ち、ROIベースで判断
- **スケーラビリティ**: 1法人契約で複数院分の月額収益

**代表的な医療法人例**:
- 医療法人社団翔舞会（エムズ歯科、10院以上）
- 医療法人真摯会（10院以上）
- スワン歯科グループ（10院以上）
- 各地域の5〜10院展開法人

### 3.3 サブターゲット：Tier 3（2院展開）

**狙い方**: Tier 1-2で成功事例を確立後、ライト版で水平展開
- 月額¥30,000〜¥50,000の低価格帯
- レセコンCSVインポートのみでスタート可能
- ダッシュボード中心のセルフサービス型

### 3.4 TAM/SAM/SOM

```
TAM（潜在市場）= Tier 1-3 × 平均月額
  = 3,000法人 × ¥120,000/月 × 12ヶ月 = ¥43.2億/年

SAM（到達可能市場）= DX意欲のある法人（30%）
  = 900法人 × ¥120,000/月 × 12ヶ月 = ¥13.0億/年

SOM（獲得目標・24ヶ月）= SAMの2.5%
  = 約23法人 × ¥120,000/月 × 12ヶ月 = ¥3,312万/年
```

## 4. ペイン・課題の詳細分析

### 4.1 患者離脱・中断

**ペインの深さ: ★★★★★（最重要）**

| 指標 | 数値 | インパクト |
|------|------|-----------|
| 治療中断率 | 20〜30% | 5人に1人以上が治療途中で離脱 |
| リコール来院率 | 40〜60% | 定期検診の案内を送っても半数未満 |
| 中断による年間損失 | ¥300〜¥800万/院 | 保険+自費の逸失売上 |
| 新患獲得コスト | ¥15,000〜¥30,000/人 | Web広告・看板・紹介等 |
| 既存患者維持コスト | ¥1,000〜¥3,000/人 | ハガキ・SMS・LINE |

**現状の対応**:
- リコールハガキの一斉送付（反応率10〜15%）
- 電話での個別連絡（スタッフの時間を大量消費）
- 来なくなった患者の原因分析ができない
- 「いつ中断するか」の予兆を掴めない

**ML解決の方向性**:
- 来院履歴・治療内容・予約パターンからの**中断リスクスコアリング**
- LightGBM回帰で「次回来院までの予測日数」を算出
- リスクスコアに応じたアプローチの自動振り分け（LINE/電話/DM）

### 4.2 予約管理・ノーショー

**ペインの深さ: ★★★★☆**

| 指標 | 数値 | インパクト |
|------|------|-----------|
| ノーショー率 | 5〜10% | 1日2〜3枠の空きが発生 |
| 直前キャンセル率 | 8〜15% | 代わりの患者が入らない |
| 1枠あたりの機会損失 | ¥5,000〜¥15,000 | 保険+自費の加重平均 |
| 年間ノーショー損失 | ¥150〜¥400万/院 | 稼働率低下の累積 |

**課題**:
- 予約リマインドはあるが「誰がキャンセルしそうか」は分からない
- 予約枠の設計が経験則に依存（30分枠 vs 60分枠の最適化なし）
- キャンセル待ちの仕組みが手動
- 天候・曜日・時間帯によるキャンセル率の変動を把握できていない

**ML解決の方向性**:
- 患者属性・履歴・天候からの**ノーショー確率予測**
- 高リスク予約への自動リマインド強化（前日+当日朝）
- キャンセル発生時のダイナミック予約枠管理

### 4.3 自費率向上・収益最大化

**ペインの深さ: ★★★★★**

```
歯科医院の収益構造
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
保険診療:      レセプト単価 ¥7,000、利益率 10〜20%
自費診療:      単価 ¥30,000〜¥500,000、利益率 60〜80%
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

自費率15% → 年商¥5,000万（個人院平均）
自費率30% → 年商¥7,000万（＋¥2,000万）
自費率40% → 年商¥9,000万（＋¥4,000万）

→ 自費率10%アップで年商¥1,500〜2,000万増
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**課題**:
- 自費提案のタイミングと方法がドクター個人の裁量に依存
- 患者の経済状況・関心度を事前に把握できない
- 自費カウンセリングの成約率が低い（20〜30%）
- どの自費メニューをどの患者に提案すべきか体系化されていない

**ML解決の方向性**:
- 患者属性・治療履歴・過去自費歴からの**自費受容確率予測**
- 最適な自費メニューのレコメンド
- カウンセリング成約率の予測と改善要因の特定（SHAP）

### 4.4 人手不足・生産性

**ペインの深さ: ★★★★☆**

| 指標 | 数値 | インパクト |
|------|------|-----------|
| 歯科衛生士有効求人倍率 | 20倍以上 | 採用が極めて困難 |
| 歯科衛生士離職率 | 1年目30%、5年目50% | 育成コストの回収困難 |
| 平均年収（歯科衛生士） | 約380万円 | 人件費上昇圧力 |
| 診療以外の業務割合 | 25〜35% | 受付・電話・事務に時間消費 |

**課題**:
- 歯科衛生士の採用難が深刻、1人退職で月商100万円減少のリスク
- チェアタイムの最適化ができず、待ち時間クレームが発生
- ベテラン衛生士のスケーリング技術・患者対応ノウハウが継承されない
- 分院間でのスタッフ配置が院長の感覚に依存

**ML解決の方向性**:
- 時間帯別来患数予測 → 最適シフト自動生成
- スタッフ別生産性分析（チェアタイム効率・自費提案成功率）
- 離職リスク予測（勤怠パターン・残業時間からのアラート）

### 4.5 経営分析・売上予測

**ペインの深さ: ★★★★☆**

**現状**:
- 月次の経営指標はレセコンの集計機能で把握（過去の記録のみ）
- 来月・来期の売上予測は院長の経験則
- 保険改定・新メニュー導入時の影響シミュレーションができない
- 分院間のKPI比較が手動（Excel集計に月2〜3日）

**歯科特有のデータ特性**:
- 保険診療はレセプトデータが正確（月次・日次で粒度が細かい）
- 自費診療はカウンセリング→成約のファネルデータが必要
- 季節変動：年末年始前（12月）、学校検診後（6〜7月）がピーク
- 曜日変動：土曜が最も混み、水曜・木曜が比較的空く

**ML解決の方向性**:
- 日次/週次売上予測（保険+自費の内訳別）
- 分院別ベンチマーク・異常検知
- 保険改定シミュレーション（点数変更の影響予測）

### 4.6 データ基盤・デジタル化の遅れ

**ペインの深さ: ★★★☆☆**

```
歯科業界のデータ分断状況
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  レセコン         電子カルテ       予約システム
  (Dentis等)      (各社独自)       (Apotool等)
       │              │              │
       └──── 一部連携 ────┘              │
                  │                     │
                  └──── 連携なし ────────┘
                         │
              ┌──────────┴──────────┐
              │ 経営分析は手動集計   │
              │ → 予測分析は不可能  │
              └─────────────────────┘
```

**歯科が美容より有利な点**:
- レセコンの普及率が高い（90%以上）→ 構造化データが既に存在
- 保険請求のためにデータ入力が義務化 → データの網羅性が高い
- 患者ID（保険証番号）でデータ統合が比較的容易

**残る課題**:
- レセコンベンダーごとにデータ形式が異なる
- 自費診療のデータがレセコンに入っていないケースが多い
- 画像データ（レントゲン・口腔内写真）とテキストデータの統合が困難
- API公開しているレセコンベンダーが少ない

**ML解決の方向性**:
- レセコンCSVエクスポートからの自動データ取込
- 主要レセコン形式対応のパーサー開発
- 自費データの手動入力テンプレート提供

## 14. 結論・推奨アクション

### 14.1 セグメント評価の結論

歯科クリニックは**Phase 2での参入推奨セグメント**（スコア3.8/5.0）。美容業界（§12D: 3.7/5.0）よりわずかに高く、データ品質・投資意欲の面で優位。

**参入すべき理由**:
1. **高品質データ**: レセコン普及率90%以上、保険請求データが構造化済み
2. **深いペイン**: 治療中断率20〜30%、自費率向上が経営の最重要課題
3. **技術適合性**: LightGBM回帰が来院予測・自費受容予測に直接適用可能
4. **既存SaaSとの補完**: レセコン・予約管理の「上位レイヤー」として共存可能
5. **美容とのシナジー**: 開発資産の60%を流用可能、並行開発で効率化
6. **低い解約率**: 医療法人は一度導入すると変更しにくい（月次解約率2.5%予測）

**注意すべき点**:
1. **医療データ規制**: 匿名化・セキュリティ対策の徹底が必須
2. **信頼性の確保**: 医療分野での実績が求められる
3. **レセコン連携**: ベンダー間のデータ形式差異への対応
