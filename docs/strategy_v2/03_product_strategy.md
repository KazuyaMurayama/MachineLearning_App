## 3.2 キラー機能: 逆SHAPエンジン

### コンセプト

```
通常SHAP:  予測結果 → 要因分析（Why?）
逆SHAP:    目標設定 → 改善アクション提案（How?）
```

**これが「これなしでは仕事できない」機能になる理由:**
- 予測だけなら他ツールでもできる
- 「具体的に何をすべきか」まで教えてくれるツールは**存在しない**
- 経営判断に直結するアクショナブルな情報

### 業界別の逆SHAP活用例

| 業界 | 質問 | 逆SHAPの回答 |
|------|------|-------------|
| 製造業 | 見積を10%下げるには？ | 材質変更で¥180K削減、ロット増で¥120K削減 |
| 物流業 | 来月の受注を安定化するには？ | 大口顧客Aへの事前確認が最も効果的 |
| 不動産 | 空室を1ヶ月短縮するには？ | Wi-Fi設置(¥50K)がROI最大 |

## 3.3 AHA Moment と Time to Value

### AHA Moment（価値実感の瞬間）

| ユースケース | AHA Moment | 到達時間目標 |
|-------------|-----------|-------------|
| 製造業 | 「ベテランの見積と3%しか違わない！」 | **15分** |
| 物流業 | 「先月のデータで来月を予測したら実際と一致！」 | **20分** |
| 不動産 | 「空室が長い物件の理由がSHAPで明確に！」 | **10分** |

### Time to Value: 60分 → 5分に短縮

```
0:00 ランディングページにアクセス
0:30 「無料で試す」クリック
1:00 業界選択（製造業/物流/不動産）
1:30 サンプルデータで自動デモ開始
3:00 SHAP分析結果を閲覧 → 「なるほど！」
4:00 「自分のデータで試す」ボタン
5:00 自社CSVアップロード → 即座に結果表示
```

**短縮施策:**

| 施策 | 短縮効果 |
|------|---------|
| サンプルデータプリセット | -30分 |
| AI推奨目的変数（カラム名・分布から自動推定） | -10分 |
| ワンクリック学習（設定スキップ） | -5分 |
| リアルタイムプレビュー | -5分 |

## 3.4 技術的差別化

### ポジショニングマップ

```
              説明可能性 高い
                   │
     当アプリ ★    │    ● 統計ソフト（SPSS等）
                   │
                   │
─────────────────┼─────────── 操作の簡単さ
    ● DataRobot    │    ● ChatGPT
                   │
    ● AWS          │
     SageMaker     │
              説明可能性 低い
```

### LightGBM + SHAP の3層差別化

| 層 | 内容 | 競合優位性 |
|----|------|-----------|
| **精度層** | LightGBM = 構造化データ最強。Kaggle上位の60%以上がGBDTベース | 少量データ（中堅企業に最適）に特に強い |
| **説明層** | TreeSHAPで厳密なShapley値を高速計算 | 個別予測+グローバル重要度の両方を提供 |
| **アクション層（新）** | 逆SHAP + What-Ifシミュレーション | **唯一無二。競合にはない** |

### データ品質自動診断機能

**ユーザー離脱の最大原因: データ品質が悪く精度が出ない → 失望 → 離脱**

```
┌──────────────────────────────────────────┐
│ 📊 データ品質レポート                     │
│                                           │
│ 総合スコア: 72/100  🟡 良好（改善推奨）   │
│                                           │
│ 📋 検出された課題:                        │
│ ⚠️ '材料コード'の欠損率が35% → [自動補完]│
│ ⚠️ '単価'に23件の外れ値 → [自動クリップ] │
│ ℹ️ '部品名'のカテゴリが250種 → [グループ化]│
│                                           │
│ [すべて自動修正] [個別に確認] [無視して続行]│
└──────────────────────────────────────────┘
```

**スコアリング基準:**
- 完全性（欠損率）: 30%
- 一貫性（型の統一性）: 20%
- 正確性（外れ値率）: 20%
- 適時性（データ鮮度）: 15%
- 十分性（行数・列数）: 15%

## 3.5 プロダクト進化ロードマップ

### Phase 1: MVP → PMF（0-6ヶ月）

**目標:** 製造業で30社の有料顧客を獲得し、PMFを確認

| 機能 | 優先度 | 実装工数 | 価値 |
|------|--------|---------|------|
| CSVアップロード + 自動学習 | P0 | 完了 | ★★★★★ |
| SHAP分析 | P0 | 完了 | ★★★★★ |
| データ品質自動診断 | P0 | 2週間 | ★★★★★ |
| 業界別サンプルデータ | P0 | 1週間 | ★★★★ |
| 逆SHAP提案エンジン（基本版） | P1 | 3週間 | ★★★★★ |
| 予測信頼区間 | P1 | 1週間 | ★★★★ |
| PDF/PPTレポート生成 | P1 | 2週間 | ★★★★★ |

**PMF判定基準（品質チェック反映: 創業者効果を排除）:**
- 週次アクティブ率 > 40%
- NPS > 40
- **セルフサービス獲得顧客**のリテンション率 > 85%
- 無料→有料転換率 > 10%

### Phase 2: Growth（6-18ヶ月）

**目標:** 3業界で500社、ARR 3億円

**主要機能:**
1. **業界ベンチマーク** — 匿名化データの蓄積、他社比較
2. **API連携** — 基幹システムからの自動データ取得
3. **チーム機能** — 複数ユーザー、権限管理
4. **自動再学習** — 新データ追加時にモデル自動更新
5. **Streamlit → Next.js + FastAPI移行**

**データフライホイール:**
```
ユーザー増加 → データ蓄積（匿名化）→ ベンチマーク精度向上
    ↑                                         ↓
新規ユーザーの魅力増大 ←── ベンチマーク比較の価値向上
```

### Phase 3: Moat構築（18-36ヶ月）

**目標:** 競合が真似できない堀の構築

```
Layer 4: エコシステムMoat（パートナー網、API連携、業界標準化）
Layer 3: データMoat（業界ベンチマーク、匿名化集合知）
Layer 2: ナレッジMoat（業界テンプレート、改善パターン）
Layer 1: テクノロジーMoat（逆SHAP、ドメイン特化AutoML）
```

**分類モデル追加: Phase 2後半（12-18ヶ月目）**
- 判断基準: 有料顧客200社超 + 顧客要望30%以上
- TAM拡大効果: 1.5-2倍（+約794億円）
