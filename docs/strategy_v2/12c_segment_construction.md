## 1. エグゼクティブサマリー

**建設業は、労働力不足が最も深刻なセグメントであり、AI/ML活用の「必要性」が極めて高い。ただし業界特有のデジタル化の遅れが参入障壁となる。**

| 項目 | 評価 |
|------|------|
| 市場規模 | 国内建設市場 約32.4兆円（2025年）、事業者数 約47万社 |
| ターゲット企業数 | 従業員30〜500名の中堅建設会社 約5,000〜8,000社 |
| ML/DS関連ペインの深刻度 | ★★★★★（労働力不足・原価管理・安全管理が喫緊の課題） |
| IT投資意欲 | ★★★☆☆（BIM義務化で加速中だが、DXリテラシーは低い） |
| 当社ソリューションとの適合度 | ★★★★☆（原価予測・工期予測に強い適合、安全/品質は要カスタマイズ） |
| 競合強度 | ★★☆☆☆（中堅建設向けML SaaSはほぼ空白） |
| 想定ARPU | 月額 8万〜20万円（ベース＋アプリアンロック） |
| 参入優先度 | **#3（中優先・Phase2で展開）** |

**選定理由のサマリー**:
- 有効求人倍率4.6倍（全業種最高水準）の深刻な労働力不足 → AI活用は「生存戦略」
- i-Construction 2.0 / BIM義務化（2026年〜段階的）で政府がDXを強力推進
- 「建築GX・DX推進事業」で年間65億円の補助金予算 → 顧客の導入障壁低下
- 工事原価予測・工期予測はLightGBM回帰予測の適合領域
- 中堅建設会社向けのML分析SaaSは競合がほぼ不在（空白地帯が最も広い）
- ただし業界のデジタルリテラシーが低く、導入に時間がかかる → Phase2展開が妥当

## 2. 建設業界の概況

### 2.1 市場規模と構造

| 指標 | 数値 | 出典 |
|------|------|------|
| 国内建設市場規模 | 約32.4兆円（2025年） | IMARC Group |
| 建設市場予測（2034年） | USD 9,413億（約141兆円） | IMARC Group |
| 建設市場 CAGR | 4.15%（2026-2034） | IMARC Group |
| 建設業事業者数 | 約47万社 | 国土交通省 |
| 建設業就業者数 | 約480万人 | 総務省 |
| AI in Construction市場（グローバル） | USD 111億（2025年）→ USD 279億（2031年予測） | GMInsights |
| AI in Construction CAGR | 16.62%（2026-2031） | GMInsights |
| 有効求人倍率（建設） | 4.6倍（全業種最高） | 日銀短観（2025年6月） |

### 2.2 業界の構造的特徴

1. **重層下請構造**: 元請→1次→2次→3次下請の多段階構造が非効率の温床
2. **プロジェクト型ビジネス**: 毎回異なる現場・仕様で「一品生産」の性質
3. **超高齢化**: 就業者の約35%が55歳以上、29歳以下は約12%
4. **厳しい労働環境**: 残業上限規制（年720時間）が2024年から適用
5. **製造業比2割低い生産性**: デジタル化の遅れが主因
6. **規制の影響**: i-Construction、BIM義務化（2026年〜段階的）が急速にDXを推進
7. **「2025年デジタルの崖」**: DX未対応企業は年間12兆円損失リスク

### 2.3 DX/AI化の現状

- AI導入率は約30%（2024年時点）、大手ゼネコン主導で進行
- 中堅・中小建設会社はDX化が大幅に遅れている（紙・Excel中心）
- i-Construction 2.0: 2040年までに生産性1.5倍、人手30%削減目標
- BIM図面審査制度: 2026年春〜一部地域で開始、2029年全国義務化予定
- 「建築GX・DX推進事業」: 年間65億円の補助金がDXを後押し

## 4. ペイン・課題の詳細分析

### 4.1 労働力不足・高齢化

**深刻度: ★★★★★ | 経済インパクト: 業界全体で年間12兆円の損失リスク**

**現状の課題**:
- 有効求人倍率4.6倍 — 全業種で最も人が集まらない業界
- 就業者の約35%が55歳以上、10年以内に大量退職が発生
- 29歳以下は約12%で、若年層の入職が極端に少ない
- 残業上限規制（年720時間）が2024年から適用され、人手不足がさらに深刻化
- 熟練技能者のノウハウ（コンクリート打設、鉄筋組立等）のブラックボックス化

**経済的インパクト**:
- 人件費上昇: 建設労務単価は過去10年で約40%上昇
- 工期遅延: 人手不足による平均工期遅延 10〜20%
- AI/ロボット活用で生産性31%向上が見込まれる（2030年予測）

### 4.2 工事原価管理・コスト超過

**深刻度: ★★★★★ | 経済インパクト: プロジェクトコスト20%削減可能**

**現状の課題**:
- 建設プロジェクトの約70%が予算超過（グローバル統計）
- 見積段階での原価予測精度が低く、実行予算との乖離が大きい
- 資材価格の変動（鉄骨、木材、セメント等）への対応が後手
- 多工種にまたがる原価構成の複雑さ（直接工事費、間接工事費、一般管理費）
- 実行予算と実績の比較分析が月次ベース（リアルタイム性なし）

**経済的インパクト（売上50億円の中堅ゼネコンの場合）**:
- 原価超過の平均: 受注額の5〜15% → 年間 2.5億〜7.5億円の利益損失
- AI原価予測で超過率を半減 → **年間 1.25億〜3.75億円の改善**
- AI導入で建設コスト20%削減の事例あり

### 4.3 工期予測・工程管理

**深刻度: ★★★★☆ | 経済インパクト: 工期遅延の半減**

**現状の課題**:
- 天候・資材納期・人員確保等の不確実要因が多く、工期予測が困難
- クリティカルパスの管理が属人的（ベテラン工程管理者に依存）
- 複数現場を並行管理する際の最適化ができていない
- 工期遅延 → 違約金、信用低下、次案件への影響

**データ分析による改善**:
- 過去工事データから工期の変動要因を特定（天候、工種、規模、地域等）
- 工種別の生産性データに基づく精度の高い工程計画
- What-if分析（「雨天が3日続いたら工期はどうなるか？」）

### 4.4 安全管理・事故防止

**深刻度: ★★★★★ | 経済インパクト: 事故1件あたり数百万〜数億円**

**現状の課題**:
- 建設業の死亡災害は全産業の約30%を占める
- 墜落・転落、重機との接触、崩壊が3大事故要因
- KY（危険予知）活動は紙ベースの形骸化が進行
- 現場の安全管理は監督者の経験に依存

**成功事例**:
- 鹿島建設「K-SAFE」: 6.4万件の災害事例をAI解析、危険予知を自動化
- 大成建設: AIカメラで人・障害物を検知し、重機接触事故を防止
- 清水建設: AI活用で山岳トンネル工事の待機時間40%削減

### 4.5 品質管理・検査効率化

**深刻度: ★★★★☆ | 経済インパクト: 検査時間60%短縮**

**現状の課題**:
- 出来形検査・配筋検査・コンクリート打設管理等が目視・手作業中心
- 検査記録の作成に膨大な時間（工事写真整理含む）
- 品質不良の発見が遅延 → 手戻り工事コスト増大
- BIM/CIM連携した品質管理への移行が必要

**成功事例**:
- CONSAIT: ゼネコン21社共同開発、AI画像検査で検査時間60%短縮
- 大成建設: AI画像認識でコンクリート打継面を1秒で判定

### 4.6 データ基盤・デジタル化の遅れ

**深刻度: ★★★★☆ | 障壁としての影響度: 非常に高い**

**現状の課題**:
- 中小建設会社の大多数はExcel・紙での管理が主流
- 工事データが案件ごとにサイロ化（統合データベースなし）
- IoTセンサー・ドローンのデータ活用基盤がない
- IT人材が社内にいない（IT兼任の総務担当レベル）
- 「何をデジタル化すべきかわからない」企業が多い

**当社にとっての意味**:
- **最大の参入障壁であると同時に最大のビジネス機会**
- データ整備から伴走支援することで長期的な関係を構築可能
- ただしデータ整備に時間がかかるため、ROI実感までのリードタイムが長い → Phase2が適切

## 6. 求められるソリューション

### 6.1 工事原価予測AI

**ニーズの強さ: ★★★★★ | 当社対応可能度: ★★★★★**

| 要素 | 詳細 |
|------|------|
| 求められる機能 | 工事案件の概算・実行予算の精度向上、コスト超過リスク予測 |
| データ入力 | 過去工事データ（工種、規模、地域、工期、資材単価、労務費） |
| 出力 | 予測原価、コスト超過確率、主要変動要因（SHAP） |
| 当社アプリ対応 | **LightGBM回帰予測アプリがそのまま活用可能** |

**顧客が求める要件**:
1. 「なぜコストが超過するのか」の要因分析（SHAP）
2. 見積段階での概算精度向上（±10%以内）
3. 資材価格変動のシナリオ分析

### 6.2 工期・工程最適化

**ニーズの強さ: ★★★★☆ | 当社対応可能度: ★★★★☆**

| 要素 | 詳細 |
|------|------|
| 求められる機能 | 工期の予測、天候・リソース制約を考慮した工程計画 |
| データ入力 | 過去工事実績、気象データ、人員配置、資材納期 |
| 出力 | 予測工期、遅延リスクスコア、クリティカルパス分析 |
| 当社アプリ対応 | 回帰予測アプリ + 要因分析（SHAP）で対応 |

### 6.3 安全リスク予測

**ニーズの強さ: ★★★★★ | 当社対応可能度: ★★★★☆**

| 要素 | 詳細 |
|------|------|
| 求められる機能 | 工事種別・現場条件に基づく事故リスク予測 |
| データ入力 | 過去事故データ、現場条件、天候、工種、作業員属性 |
| 出力 | リスクスコア、高リスク要因TOP5、推奨安全対策 |
| 当社アプリ対応 | 分類モデル（事故有無予測）+ SHAP解析 |

**参考**: 鹿島建設K-SAFEは6.4万件の災害事例をAI解析 → 当社は同様のアプローチをSaaS化

## 8. 当社ポジショニング・差別化

### 8.1 ポジショニングステートメント

> **中堅建設会社**に対して、
> **IT部門不在でも使える、工事原価・工期・安全のML予測ツール群**を、
> **月額8万〜20万円のSaaS＋伴走支援**で提供する、
> **「建設版 一人DSチーム」プラットフォーム**

### 8.2 差別化ポイント（5つ）

| # | 差別化要素 | 詳細 | 競合との違い |
|---|-----------|------|-------------|
| 1 | **建設業×ML予測の唯一のSaaS** | 原価・工期・安全を予測モデルで | ANDPAD等との差別化 |
| 2 | **SHAP解析標準搭載** | 「なぜコスト超過するか」を可視化 | ブラックボックス型との差別化 |
| 3 | **CSV入力で即分析** | BIM連携不要、Excel工事台帳から開始可能 | BIM前提のシステムとの差別化 |
| 4 | **補助金活用支援** | 建築GX・DX推進事業の申請サポート | ツール売り切り型との差別化 |
| 5 | **他セグメントとの共通基盤** | 食品製造業・EC向けアプリの知見を活用 | 建設業単一特化との差別化 |

### 8.3 ANDPAD等との補完関係

既存の建設DX SaaS（ANDPAD、SPIDERPLUS等）は「施工管理のデジタル化」に注力しており、ML予測・分析は提供していない。当社はこれらのSaaSと**補完関係**を構築可能:

- ANDPAD/SPIDERPLUSでデジタル化されたデータ → 当社アプリでML分析
- 施工管理SaaSからCSVエクスポート → 当社アプリにアップロード → 予測・分析結果を返す
- 将来的にはAPI連携でシームレスな統合も可能

## 13. 成功事例・ケーススタディ

### 13.1 鹿島建設「K-SAFE」
