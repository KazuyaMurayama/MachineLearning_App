# UX品質エージェントによる評価

> 評価者: UX Critic Agent
> 日付: 2026-04-08

## エグゼクティブサマリー

TASK 1-2の静的構文チェック（ast.parse）は「コードが壊れていないこと」を確認したにすぎず、実際のブラウザ体験を一切検証していない。ポータル2本のFirst Impressionは3秒以内に「誰向け・何ができるか」を伝えるという基準でおおむね合格だが、L3差別化の視覚表現が弱く、価格訴求が埋もれている。17アプリ全体のブランド統一性は配色レベルでは保たれているが、タブ構成・サイドバー設計に個体差があり、デモ文脈での一貫感に課題がある。

---

## TASK 1-2 の UX 盲点

### 静的チェックでは見えないUX問題

`ast.parse()` はPython構文の整合性を確認するツールであり、以下のUX要素はゼロ検証である。

- **レンダリング遅延**: `@st.cache_resource` で LightGBM 3モデルを初回ロードするec-what-ifは、コールドスタートで数十秒かかる可能性がある。ユーザーはスピナーを見続けるが、静的チェックはこれを検知しない。
- **フォント欠落**: shigyou-office-dashboardはNoto Sans CJK JPなどを動的サーチし、なければ`sans-serif`にフォールバックする。Streamlit Cloudの実行環境によっては日本語グラフラベルが文字化けする。`ast.parse()`はこれを見逃す。
- **カラムの縦崩れ**: `st.columns(4)` のKPIカードはモバイル幅（< 768px）で強制縦積みになり、デザイン意図が崩壊する。
- **クリック不能カード**: HTMLアンカーで包んだカードはStreamlit iframeのサンドボックスポリシーに依存する。`target="_blank"`が機能しない環境では何も起きない。

### 未検証の実体験

「動く」は確認済み。しかし以下は一切未確認。

- 初見ユーザーが「月額30万円」のL3バナーを見て納得するまでの心理フロー
- スライダー操作後のリアルタイム予測レスポンスタイム（ec-what-if）
- サイドバー展開時にメインコンテンツが押しつぶされる表示崩れ
- デモ進行中に「データ再生成」ボタンを押した際のリグレッション

## ポータル 2本の UX評価

### streamlit_app.py（士業）

- **First Impression**: Heroセクション1行目「士業AI経営パートナー」で対象者は伝わる。2行目「AIで変革する11の専用ツール」で機能量も伝わる。3秒判定は合格。ただし `font-size: 2.4rem` はスマホ（320px幅）で折り返しが2回以上発生し、Heroの「3秒で読める」特性が失われる。
- **L3セクションの差別化**: `l3-section-banner` に紺→紫のグラジエントを充てているが、他カードとの差は「バナー1本」のみ。カード自体は白背景の同一デザインで、L3と非L3の視覚ウェイト差がほぼない。価格「月額30万円」はサイドバーの太字テキストに埋まっており、メインコンテンツ上では一切表示されない（バナー本文は競合比較の文脈で登場するが、数字が出てこない）。
- **問題点**: (1) 価格が見えない、(2) L3カード3枚と非L3カード5枚が同じ視覚強度、(3) KPIカード「80%削減」「年150h+」はデータソースなしで信頼性が低い、(4) 導入ステップのStep 1-3が下部に埋もれCTAとして機能していない。

### ec_app.py（EC）

- **First Impression**: Heroの「EC AIマーケティングパートナー」は業態は伝わるが「誰向け」がやや広い。サブコピー「顧客分析から広告最適化まで」は機能列挙で感動がない。3秒判定は条件付き合格。
- **L3セクションの差別化**: バナーに「GA4・Lookerでは見えない"次の一手"」という競合対比コピーを入れており、士業版より訴求が明確。ただしKPIカード4枚で「LTV」「ROAS」「即日」と並んでいるのは抽象的すぎる。数値（例: ROAS +30%）を入れるべき場所に概念語が入っている。
- **問題点**: (1) L1/L2セクションの見出しが「L1/L2 コアツール — 4ツール連携で…」と長すぎる、(2) Step 4カードが3カラムの残り2枠を空けて孤立表示され、グリッドが崩れる、(3) ec-demo の`https://ec-demo.streamlit.app`はURLが汎用的すぎてデモ中に混乱を招く可能性がある。

## L3 6アプリの UX評価（簡易）

| アプリ | デモ映え | 課題 |
|--------|---------|------|
| shigyou-office-dashboard | 中：KPIカード+フィルタで業務感は高い | 日本語グラフフォントの環境依存リスク、サイドバー占有率が高くメインが狭い |
| shigyou-ltv | 高：SHAP beeswarmは視覚的インパクト大。不採算フラグの赤表示は感情を動かす | モデル学習待機中のUI（スピナーのみ）が地味、進捗バーがない |
| shigyou-briefing | 低：テキスト出力中心でビジュアルがない | Markdownレポートはコピペ用途には便利だが画面映えしない。デモで「おっ」という瞬間がない |
| ec-executive-dashboard | 中：横断ROI分析は説明力が高い | 在庫アラートや離脱リスクTOP10の視覚化強度が不明（未読部分） |
| ec-what-if | 最高：スライダー操作→リアルタイム3モデル予測は唯一無二のデモ山場 | コールドスタートの数十秒ロードがデモの流れを断ち切るリスク |
| ec-monthly-briefing | 低：shigyou-briefingと同構造の疑い。テキスト＋MDエクスポート | 「月額26万円の伴走支援を可視化」というコピーの期待値に対して画面が地味 |

## 最も改善インパクトが大きい UX改善 TOP5

1. **ec-what-if のウォームアップ表示**: モデル学習中に「LightGBM 3モデル学習中…売上モデル完了 1/3」のような段階的プログレスを表示する。コールドスタートの沈黙はデモ致命傷であり、改善コストは低い（`st.progress` + `st.status` 追加のみ）。

2. **ポータル2本のL3価格を可視化**: メインコンテンツのL3バナー内に「月額30万円（DS採用比 1/3）」「月額26万円（DataRobot比 1/4）」を大きく表示する。現状はサイドバーの太字のみで埋もれており、価格訴求がデモ相手に届いていない。

3. **briefing系2本にサマリービジュアルを追加**: 月次AIブリーフィングの冒頭に「今月のアクション優先度」をゲージ or バーチャートで1枚表示する。テキスト出力前に視覚的なキャッチを置くことでデモの「おっ」ポイントを作る。

4. **モバイル対応: st.columns(4) をst.columns([1,1]) に分割**: KPIカード4枚を2行×2列の入れ子columnsに変更し、スマホ・タブレット縦持ちでの崩れを防ぐ。タブレットデモが想定される場合、これは必須の対応。

5. **17アプリ横断のサイドバー統一**: shigyou-office-dashboardは業種/担当者フィルタがサイドバーを占拠する。他アプリと設計方針が異なり、デモ中に「このアプリだけ操作が違う」という混乱を生む。フィルタをメインエリアのexpanderに移し、サイドバーはナビゲーション専用に統一する。

## デモシナリオ設計への示唆

15分デモの推奨順序（感情の山を意識した構成）

| 時間 | 画面 | 狙い |
|------|------|------|
| 0:00〜1:00 | 士業ポータル（streamlit_app.py） | 「11本ある」という全体感を1画面で把握させる。価格を口頭で補足 |
| 1:00〜3:00 | shigyou-ltv | SHAP beeswarmで「不採算顧問先がAIで見える」インパクトを与える。感情の第1ピーク |
| 3:00〜5:00 | shigyou-office-dashboard | KPIカード+フィルタ操作でダッシュボードの業務感を示す。地に足のついた実用性 |
| 5:00〜6:00 | shigyou-briefing | 「月次レポートが5分で出る」実演。地味だが時間節約の価値を数字で示す |
| 6:00〜7:00 | ECポータル（ec_app.py） | 士業→EC切替でターゲット層の広さをアピール。ポータル設計の統一感も見せる |
| 7:00〜12:00 | ec-what-if | スライダー操作→リアルタイム3モデル予測。感情の最大ピーク。必ずここに時間を厚く割く |
| 12:00〜14:00 | ec-executive-dashboard | 横断ROI+在庫アラートで「経営ダッシュボードとセットで使う」文脈を作る |
| 14:00〜15:00 | クロージング | 価格比較（DataRobot・DS採用・船井総研）を画面ではなくスライドで提示。ポータルのフッターに価格を載せておくと補助になる |

**注意**: ec-what-ifはコールドスタートに備え、デモ開始15分前にブラウザで事前ロードしておくこと。shigyou-briefingとec-monthly-briefingはデモ後半に配置し「おっ」ポイントの間に挟まないことで地味さを目立たせない。
