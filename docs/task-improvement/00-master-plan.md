## Phase 1: マルチエージェント独立評価

5つの異なる思考モードのエージェントが TASK 1-3 をそれぞれ独立に評価する。

| Agent | 役割 | 視点 | 出力ファイル |
|-------|------|------|------------|
| **A: 批判的思考** | Critical Thinker | 前提の疑い、見落とし、楽観バイアス | `01-critical-thinking.md` |
| **B: 論理的思考** | Logical Analyst | 論理矛盾、根拠不足、一貫性 | `02-logical-analysis.md` |
| **C: ビジネス戦略** | Strategy Analyst | ROI、市場適合、訴求力、収益性 | `03-business-strategy.md` |
| **D: UX品質** | UX Critic | 初見体験、UI一貫性、操作性 | `04-ux-quality.md` |
| **E: 技術レビュー** | Tech Reviewer | コード品質、パフォーマンス、保守性 | `05-tech-review.md` |

### 各エージェントの評価フレームワーク

各タスク（1-3）について以下を出力:
1. **発見した問題点** (🔴重大 / 🟡中程度 / 🟢軽微)
2. **見落としリスク** (デプロイ後に顕在化しそうな問題)
3. **推奨改善策** (具体的なアクション)
4. **改善による期待効果** (定量的に)

---

## Phase 2: 相互レビュー・統合

Phase 1の5つの評価結果を統合エージェントが読み込み、以下を実施:
- **合意点**: 複数エージェントが指摘した共通問題 → 最優先
- **異議点**: エージェント間で意見が対立する箇所 → 追加検討
- **盲点**: どのエージェントも指摘していないが重要な要素

出力: `06-synthesis.md`

---

## Phase 3: アクションプラン

統合結果から優先順位付き改善タスクを策定:

- **P0**: 緊急（ブロッカー・重大な漏れ）
- **P1**: 高（訴求力・品質に大きく影響）
- **P2**: 中（あると良い）
- **P3**: 低（将来対応）

各タスクに対して:
- 工数見積 (S/A/B)
- 期待効果
- 依存関係

出力: `07-action-plan.md`

---

P0とP1タスクを順次実行。各ステップ完了ごとにファイルを保存。
