# 料金およびスコープ定義書（Pricing & Scope）

本書は、MachineLearning_App（以下「本サービス」）の料金体系、プラン構成、提供範囲、SLA の概要を一元管理する **唯一の情報源（SSOT）** です。提案書・契約書・見積書を作成する際は、必ず本書の最新バージョンと整合していることを確認してください。

> **戦略整合**: 本書の料金は上流リポ `freelance-compass` の最新戦略 [integrated-business-plan-v2.md](https://github.com/KazuyaMurayama/freelance-compass/blob/main/outputs/integrated-business-plan-v2.md)（2026-03-25）の 3 層モデル（L1/L2/L3）に整合させています。本書「スタンダード」プランは v2 の L3（AI経営パートナー）相当です。

> **重要: サービス境界に関する注記**
> 本サービスは経営意思決定の **補助** を目的とするものであり、税務判断（税理士法52条）、法律判断（弁護士法72条）、会計監査判断に代替するものではありません。顧客がこれらの専門判断を必要とする場面では、有資格者への相談を推奨してください。

## 1. プラン概要

| プラン | 対象フェーズ | 期間 | 主用途 | v2 対応層 |
| --- | --- | --- | --- | --- |
| パイロット | 導入検証 | 6週間固定 | L3アプリのPoC、効果測定、継続判断 | L3 の前段 |
| スタンダード | 本番運用 | 12か月契約（自動更新） | 日常業務への組み込み、月次KPI運用 | **L3** |
| エンタープライズ | 全社展開 | 12か月契約（要個別設計） | 複数拠点、カスタムモデル、専任体制 | L3 拡張 |

> v2 の L1（AI業務効率化、¥50,000/月）および L2（ML予測モデル、士業 ¥100,000 / EC ¥120,000 月額）は現行のパイロット／スタンダードには直接対応していません。L1/L2 相当のスポット提供・単発コンサルは §4 追加オプションで提供可能です。

## 3. 料金に含まれるもの・含まれないもの

### 3.1 パイロットプランに含まれるもの

- L3アプリ（業種別3アプリ）のホステッド環境提供
- 初期データ取り込み支援（CSV / API、最大2データソース）
- キックオフミーティング（1回、90分）
- 週次進捗ミーティング（6回、各45分）
- 中間レビュー（1回、60分）
- 最終レビューと継続判断ミーティング（1回、90分）
- AIブリーフィングの初期プロンプト調整（2回まで）
- 平日9:00〜18:00のメールサポート

### 3.2 パイロットプランに含まれないもの

- 顧客固有データを用いた予測モデルの再学習・カスタマイズ
- 基幹システムとのAPI連携開発
- オンプレミス環境へのデプロイ
- 24時間365日のサポート
- 訪問対応（オンサイト）
- セキュリティ監査対応資料の個別作成（簡易版はスタンダード以降で提供）

不明確な境界が発生した場合は、Sales Opsに確認のうえ「スコープ外」または「追加オプション」として明示してください。

## 5. SLA概要

本SLAはスタンダード以降に適用されます。パイロットプランは「ベストエフォート提供」とし、法的なSLA保証の対象外です。ただし、パイロット期間中の重大障害時は1営業日以内に状況報告を行います。

| 項目 | 目標値 |
| --- | --- |
| 月間稼働率 | 99.5%以上（仮目標、要検証） |
| 重大障害時の一次応答 | 2営業時間以内 |
| 通常問い合わせの一次応答 | 1営業日以内 |
| サポートチャネル | メール、チャット（平日9:00〜18:00） |
| 計画メンテナンス窓 | 毎月第2日曜 01:00〜05:00（JST） |
| メンテナンス事前通知 | 5営業日前まで |

予測モデルの精度に関するSLAは現時点では設定していません。

## 6. 支払条件・請求サイクル

- **パイロット**: 開始前に全額請求（前払い）。振込期限は請求書発行日より14日以内。※初期5社向けに **分割払い（開始時50%／完了時50%）** も選択可能。
- スタンダード: 月次請求、翌月末払い。
- エンタープライズ: 四半期または年次の前払いを推奨。個別交渉可。
- 支払方法: 銀行振込（原則）。
- 通貨: 日本円（JPY）のみ。消費税は別途申し受けます。

## 8. パイロット成功基準・KPI・転換目標

### 8.1 パイロット合格基準（3項目中2項目以上達成で「合格」）

1. **主力機能の日常利用**: ダッシュボードまたはAIブリーフィングが週1回以上参照される
2. **アクション起動**: パイロット期間中に少なくとも1件の具体的な業務改善アクションが開始される
3. **顧客満足**: 最終レビューでNPS 7以上（10点満点）

### 8.2 事業KPI目標（仮説、5社完了後に更新）

| 指標 | 目標値 | 測定方法 |
| --- | --- | --- |
| パイロット→スタンダード転換率 | 50%以上（🔸仮説） | 契約件数 / パイロット完了件数 |
| パイロット顧客獲得コスト（CAC） | ¥100,000以下（🔸仮説） | 営業活動総コスト / パイロット契約件数 |
| 年間LTV（スタンダード） | 士業¥2,400,000 / EC¥3,000,000 | 月額 × 12か月 |
| CAC回収期間 | 1か月以内 | CAC / 月額 |

### 8.3 失敗シナリオと対応

| シナリオ | トリガー | 対応 |
| --- | --- | --- |
| パイロット途中離脱 | 顧客が中間レビュー後に中止要請 | 離脱理由ヒアリング → 改善バックログへ → 未提供分の返金検討 |
| 5社中0社転換 | 全社がスタンダード移行しない | プロダクト・価格・対象ICP の全面見直し → ピボット判断 |
| モデル精度が改善しない | パイロット完了時にR²<0.3 | 該当機能を「実験的」表記に変更 → スタンダードからスコープ外化も検討 |
