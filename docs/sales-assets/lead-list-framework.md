# リードリスト フレームワーク

## 1. 背景と目的

現状、L3プロトタイプは完成しているにもかかわらず、リード獲得戦略が体系化されておらず、パイロット顧客も0社です。本書は、明日から動けるリード獲得フレームワークを提供することを目的としています。

目標は次のとおりです。

- **2週間以内**: 10社の見込リストを作成（名刺交換済みまたは紹介依頼済みの状態）
- **4週間以内**: 5社とパイロット契約締結

短期スプリントであるため、本書は「広げる」よりも「絞って動く」ことに重点を置いています。

## 2. 4チャネル戦略

リード獲得は以下4チャネルで構成します。現時点では既存ネットワーク依存の比率が高くなります。

### チャネル1: 既存ネットワーク・紹介

- **想定比率**: 全リードの60%（🔸仮説）
- **アクション手順**:
  1. 担当者個人のLinkedIn、名刺管理ツール、個人メール、SNSから過去接点を洗い出す
  2. 士業・EC事業者に該当する接点を抽出
  3. 温度感の高い順に並べ、1日5件を目安に「6週間パイロットの5社を探している」文脈で連絡
  4. 面談設定できたものはICP判定しリード台帳に記載
- **想定歩留まり（🔸仮説）**: 接触50件 → 面談15件 → パイロット合意3件

### チャネル2: 業界団体・勉強会

- **想定比率**: 全リードの15%（🔸仮説）
- **アクション手順**:
  1. 士業向けの研究会・青色申告会・地域会計人団体等の公開勉強会に登録
  2. EC事業者向けの実務者コミュニティ（クローズド系）に参加申請
  3. 登壇機会があれば優先して受ける（参加者より発信者の方が信頼形成が速い）
  4. 懇親会・チャット・Q&Aで個別対話に展開
- **想定歩留まり（🔸仮説）**: 接触20件 → 面談6件 → パイロット合意1件

### チャネル3: オンライン発信

- **想定比率**: 全リードの15%（🔸仮説）
- **アクション手順**:
  1. 担当者名義のnote/X/LinkedInで、士業・EC向けの意思決定の話題を週2本のペースで発信
  2. プロトタイプのスクリーンショットではなく、「考え方」を発信（機密配慮）
  3. 反応のあった人にはDMで30分の対話を打診
  4. 営業色の強い投稿は避け、比率は「学び9:告知1」
- **想定歩留まり（🔸仮説）**: インプレッション非数値、DM返信10件 → 面談4件 → パイロット合意1件

### チャネル4: 限定的なアウトバウンド

- **想定比率**: 全リードの10%（🔸仮説）
- **アクション手順**:
  1. ICPに合致する企業を公開情報のみから最大20社リストアップ
  2. 代表電話でのコールドコールは実施しない
  3. 公開されているお問い合わせフォーム、または代表者の公式SNSに限定して、相手に敬意を持った1通の連絡を行う
  4. 返信がなければ追撃は1回のみ
- **想定歩留まり（🔸仮説）**: 20件 → 面談2件 → パイロット合意0〜1件
- **注意**: 個人情報の外部リストは使用しない。アンチパターン参照。

## 3. ICP（Ideal Customer Profile）定義

### 3.1 士業ICP

- **規模**: 職員5〜30名の税理士・会計士事務所
- **地域**: 全国対象。ただし初期は対面可能な地域を優先
- **主要顧客層**: 中小企業クライアント中心（個人事業主特化型は除外）
- **ITリテラシー**: クラウド会計を導入済み、またはチャットツール運用経験あり
- **痛みの兆候**:
  - 「顧問先が最近離れた」話を自発的にする
  - 所長が経営相談業務を拡大したい意向を持つ
  - 所員の定着に課題感がある
- **除外条件**:
  - 職員1〜2名の零細事務所（運用工数が確保できない）
  - 紙/ファックス中心のオペレーション
  - 既に類似AIサービスに大型投資済み

### 3.2 EC ICP

- **月商レンジ**: 月商5,000万円〜10億円
- **商材**: 自社企画商品（物販）、消耗型サブスク
- **プラットフォーム**: 自社EC+モール併用型
- **組織体制**: 経営者の下にマーケ責任者が存在
- **痛みの兆候**:
  - CAC/ROASの話題が経営会議の主要議題になっている
  - 広告代理店への不満が口頭で出る
  - データを見たいが時間が取れないと発言する
- **除外条件**:
  - D2C立ち上げ直後（データ蓄積不足）
  - 取扱品目が数万SKU超（対象外のデータ構造）
  - 既にフルオーダーのBIコンサルを高額契約中

## 4. リード評価スコア（簡易BANT）

各リードを以下4軸で0〜3点で評価します。合計12点満点。**8点以上**を優先リードとします。

- **B（Budget / 予算）**
  - 0: 予算概念なし / 1: 仮検討 / 2: 確保可能な裁量 / 3: 予算枠あり
- **A（Authority / 決裁者）**
  - 0: 現場のみ / 1: 現場+影響力 / 2: 決裁者にパス可 / 3: 決裁者本人と対話中
- **N（Need / 必要性）**
  - 0: ペイン不明 / 1: ペイン認識 / 2: ペイン言語化済 / 3: ペイン解決を宣言
- **T（Timing / タイミング）**
  - 0: 未定 / 1: 半年以上先 / 2: 3か月以内 / 3: 即時検討

**運用ルール**: スコアは毎週金曜に更新。3週間連続でスコアが変わらないリードはステータスを見直します。

## 5. リード台帳テンプレート（CSV列定義）

リード管理は共有CSVまたは簡易スプレッドシートで行います。列定義は以下のとおりです。

| 列名 | 型 | 説明 |
|---|---|---|
| lead_id | string | 自動採番 L-0001 形式 |
| 会社名 | string | 正式名称（社内管理上の仮名可） |
| 業種 | enum | 士業 / EC |
| 規模 | string | 士業は職員数、ECは月商レンジ |
| 接点経路 | enum | 既存NW / 紹介 / 団体 / 発信 / アウトバウンド |
| 決裁者 | string | 役職とフルネーム（または識別子） |
| 痛みヒアリング済 | bool | Yes/No |
| BANTスコア | int | 0〜12 |
| 次アクション | string | 具体的内容 |
| 次アクション期日 | date | YYYY-MM-DD |
| ステータス | enum | 未接触 / 接触済 / 面談設定 / 面談実施 / 提案中 / 契約 / 失注 / 保留 |
| 最終接触日 | date | YYYY-MM-DD |
| 担当 | string | セールス担当者名 |
| メモ | text | 自由記述 |

## 6. 初動30日ロードマップ

### Week 1（2026-04-13〜04-19）: 準備とネットワーク棚卸し

- 月: ICP定義の社内共有、リード台帳CSVの初期化
- 火〜水: 担当者全員が個人の接点を洗い出し、台帳に登録（目標50件）
- 木: 台帳レビュー、優先10件を選定
- 金: 優先10件に連絡開始（月曜着想定の文面）

### Week 2（04-20〜04-26）: 面談設定ラッシュ

- 月〜火: Week1接触の返信対応、二次紹介依頼
- 水: 業界団体/勉強会のリストアップ、登録開始
- 木〜金: オンライン発信1本目、2本目を公開
- 週末時点KPI: 見込リスト10社達成

### Week 3（04-27〜05-03）: ヒアリング実行

- 月〜金: 設定済み面談を実施、`customer-pain-research.md` のヒアリングテンプレートに従って記録
- 週末時点KPI: 5社ヒアリング完了、うち2社は提案フェーズに移行

### Week 4（05-04〜05-10）: 提案とクロージング

- 月〜水: 提案書提出、稟議用1枚サマリ同封
- 木〜金: 社内決裁フォローアップ、契約書ドラフト共有
- 週末時点KPI: 5社パイロット契約

## 7. アンチパターン（やってはいけないこと）

以下は品質と信用を毀損するため、明確に禁止します。

1. **一斉メール送信**: 名寄せ済みリストへの一斉メール配信は、たとえ個別文面に見えても禁止。返信率の向上にも寄与しません。
2. **個人情報の外部収集**: 購入リスト、スクレイピングによる連絡先取得は禁止。法令・倫理の両面で不適切です。
3. **過度な値引き約束**: 定価の20%超の値引きを現場判断で口約束することは禁止。必ずエスカレーションしてください。
4. **実績の誇張**: パイロット実績0社の段階で「導入事例あり」と示唆する表現は禁止。むしろ「最初の5社」優位性として誠実に語ります。
5. **精度の断言**: L3のモデル精度、特に `ec-what-if` のR²については、現時点で参考値扱いであることを明示せずに断言しないでください（詳細は `objection-handling.md` D-3参照）。
6. **実名企業の内部メモ混在**: 本書および台帳には、他社名を踏まえた競合言及、個別企業の内部事情メモを記載しないでください。これらは別途アクセス制御されたメモに分離します。
7. **ヒアリング中の売り込み**: ヒアリングは聴く場です。最初の20分は提案を行わないでください。
8. **仮説数値の独り歩き**: `customer-pain-research.md` の仮説数値を、営業資料で出典なしに引用することは禁止。
