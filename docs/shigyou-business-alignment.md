- 複数予測モデル（最低3本以上）の並列稼働
- 経営ダッシュボードとして集約・可視化
- 月次レビューによるコンサル価値の提供
- 競合比較軸：データサイエンティスト正社員採用（月50-100万）より安い

## 3. 各MVPのギャップ分析

### 3.1 L2層のギャップ（shigyou-demo / crosssell）

**shigyou-demo（顧問先離反予測）**
- 充足: LightGBM回帰＋SHAP＋逆SHAP提案の骨格実装あり
- ギャップ①: 学習データが**デモ用サンプルのみ**。実顧問先データのCSV投入フローが未整備
- ギャップ②: 逆SHAP改善提案の具体性が低い（「連絡頻度を増やす」等の汎用表現のみ）
- ギャップ③: モデル精度指標（R²・AUC）の表示なし。信頼区間表示なし
- ギャップ④: freee/MF APIからの自動データ取込不可。毎回手動CSV操作が必要

**crosssell（クロスセル分析）**
- 充足: サービス利用状況の可視化、推奨ロジック、増収試算は機能する
- ギャップ①: 推奨エンジンが**ルールベース**。LightGBM等でスコアリングすれば精度と訴求力が向上
- ギャップ②: 「なぜこのサービスが推奨されるのか」のSHAP説明がない → L2訴求力が弱い

### 3.2 L1層のギャップ（report-gen / その他）

**report-gen（月次レポート）**
- 充足: 前月比・前年比・異常値検出・自動レポートが実装済みで実用的
- ギャップ①: freee/MFのCSVエクスポートが手動。API連携でL2水準に昇格可能
- ギャップ②: 顧問先別の比較・ベンチマーク機能なし（L3への橋渡しに必要）

**contract-draft / compliance-pack / doc-checker / payment-alert**
- L1水準として十分だが、4本が「独立したツール」として存在し、統合ダッシュボードがない
- L3到達には、これらをまとめる「事務所経営ダッシュボード」が必要

### 3.3 L3層のギャップ（未着手）

L3に必要な要素が現在のMVP群には存在しない：

| L3要件 | 現状 | ギャップ |
|--------|------|---------|
| 複数モデルの統合管理画面 | なし | 各アプリが独立しており横断ビューがない |
| 戦略レベルのコンサル資料自動生成 | なし | レポートは個別ツール内にとどまる |
| 経営KPIダッシュボード | なし | 月次レポートはCSV1枚対応のみ |
| 月次データレビュー支援 | なし | アドバイザリー機能が未定義 |

### 4.2 L3到達に必要なキラー機能（3本柱）

#### キラー機能①: 「事務所経営ダッシュボード」（L3コア）

現在バラバラな8アプリを横断する統合ビュー。訴求メッセージ：「8つのAIが1つのダッシュボードに集約 — 毎月の経営会議の代わりに」

```
実装要素:
  - 顧問先リスト × 離反スコア（shigyou-demo）
  - 入金遅延アラート統計（payment-alert）
  - クロスセル機会トップ10（crosssell）
  - 当月レポート生成数・異常値アラート（report-gen）
  → 全部を1画面で俯瞰し、月次コンサル資料として出力
```

難易度: A（既存モジュールの集約が主作業）

#### キラー機能②: 「顧問先LTV予測＋不採算先フラグ」（L2→L3アップセル鍵）

12f文書が指摘する「全顧問先に同じ対応をしており、収益性に差がある」ペインへの直撃。クロスセルアプリのデータと離反予測を結合し、顧問先1社ごとの**将来LTV**を算出する。

```
入力: 顧問料 × 契約年数見込み × クロスセル確率 × 離反リスク
出力: 顧問先別LTV予測値（5年間）+ VIP/成長/不採算クラスタ分類
訴求: 「不採算顧問先を特定し、改善または整理するだけで利益率5%改善」
```

難易度: A（既存LightGBMパイプラインの再利用）

#### キラー機能③: 「月次AIブリーフィングレポート」（L3コンサル価値の具体化）

L3の月額30万円を正当化する最大の根拠が「コンサルタントとして使える資料の自動生成」。毎月のデータ更新後に事務所経営者向けPDFを自動出力する。

```
内容:
  - 今月の顧問先状況サマリー（離反リスク上位5社）
  - 入金遅延状況と対応推奨
  - クロスセル機会ランキング（推定増収額付き）
  - 前月比・前年比の業績分析（report-genデータ活用）
  - 翌月の重点アクション提案（逆SHAP由来）
訴求: 「30分かかっていた月次経営会議の資料が5分で完成」
```

難易度: A（jinja2テンプレート + 既存分析モジュールの呼び出し）

- パイロット検証完了 → 「L3 production-ready」と表記
- 売上実績 1社以上獲得 → 「L3 商業稼働」と表記
- 現在は **L3プロトタイプ完成** ステージ
